# दीर्घकाळ चालणारा एजंट वर्कफ्लो

एखादे कार्य अनेक सत्रे, हँडऑफ किंवा स्पॉन केलेल्या एजंट्सपर्यंत असण्याची शक्यता असते तेव्हा हे प्लेबुक वापरा.

## लक्ष्ये

- प्रत्येक ताज्या सत्राला संदर्भ परत मिळवण्याचा एक जलद मार्ग द्या
- काम वाढवत ठेवा
- एक मोठा बदल करण्याऐवजी</zxqsep/>काम वाढवत ठेवा. अधिक कोड जोडण्यापूर्वी तुटलेली स्थानिक आधाररेखा
- पुढील सत्रावर विश्वास ठेवता येईल अशा टिकाऊ कलाकृती सोडा

## राज्य कुठे ठेवावे

- जेव्हा मानव, पुनरावलोकन बॉट्स किंवा एकाधिक टूलचेनला समान कार्य स्थितीची आवश्यकता असते तेव्हा `docs/agent-runs/<slug>/` वापरा.
- टास्क स्टेट एखाद्या वर्कस्टेशन किंवा एका टूलचेनसाठी हेतुपुरस्सर स्थानिक असेल तेव्हाच `.codex/runs/<slug>/` सारखी टूल-लोकल डिरेक्टरी वापरा.
- खाजगी फाईल मल्टी स्टेटस किंवा शेअर्ड स्टेटसमध्ये सामायिक करू नका किंवा शेअर करू नका. एजंटला नंतर त्याची आवश्यकता असेल.

## आवश्यक फाइल्स

दीर्घकाळ चालणाऱ्या कार्याच्या सुरूवातीस या फायली तयार करा:

- `feature-list.json`
- `progress.md`

`docs/agent-playbooks/templates/feature-list.template.json` आणि `docs/agent-playbooks/templates/progress.template.md` मधील टेम्पलेट्स वापरा.

प्राधान्य द्या JSON फील्डच्या छोट्या फील्डची फील्ड फील्ड न करता फील्डची फील्ड नवीन अद्यतनित करू शकते. दस्तऐवज.

## सत्र प्रारंभ चेकलिस्ट

1. `pwd` चालवा.
2. `progress.md` वाचा.
3. `feature-list.json` वाचा.
4. `git log --oneline -20` चालवा.
5. `./scripts/agent-init.sh --smoke` चालवा.
6. `pending`, `in_progress`, किंवा `blocked` असा एक सर्वोच्च-प्राथमिकता आयटम निवडा.

स्मोक स्टेप अयशस्वी झाल्यास, नवीन वैशिष्ट्य स्लाइस लागू करण्यापूर्वी तुटलेली बेसलाइन दुरुस्त करा.

## सत्राचे नियम

- एकावेळी एका वैशिष्ट्यावर किंवा टास्क स्लाइसवर कार्य करा.
- वैशिष्ट्य सूची मशीन-वाचनीय आणि स्थिर ठेवा. असंबंधित आयटम पुन्हा लिहिण्याऐवजी स्थिती, नोट्स, फाइल्स आणि पडताळणी फील्ड अपडेट करा.
- त्या आयटममध्ये सूचीबद्ध कमांड किंवा वापरकर्ता प्रवाह चालवल्यानंतर सत्यापित केलेल्या आयटमवर फक्त चिन्हांकित करा.
- बाउंडेड स्लाइससाठी स्पॉन केलेले एजंट वापरा, संपूर्ण टास्क-स्टेट मालकीसाठी नाही.
- जेव्हा लहान एजंटकडे एक आयटम असेल, तेव्हा त्याला अचूक आयटम आयडी, स्वीकृती निकष आणि तो स्पर्श करू शकणाऱ्या फाइल्स द्या.

## सत्र समाप्ती चेकलिस्ट

1. ZOLX PLACEH PLACEH PLACEHQ 0 ची लहान प्रगती जोडा.
2. `feature-list.json` मध्ये स्पर्श केलेला आयटम अद्यतनित करा.
3. सत्यापनासाठी चालवल्या जाणाऱ्या नेमक्या आदेशांची नोंद करा.
4. कॅप्चर ब्लॉकर्स, फॉलो-अप आणि पुढील सर्वोत्कृष्ट आयटम पुन्हा सुरू करण्यासाठी.

# ईव्हीएम कॉन्ट्रॅक्ट कॉल चॅलेंज

:::warning Legacy Naming
हे पॅकेज मूळत: `@plebbit` स्कोप अंतर्गत प्रकाशित केले गेले. त्याचे नाव बदलून `@bitsocial/evm-contract-challenge` असे करण्यात आले आहे. जुन्या नावाचे संदर्भ अजूनही जुन्या कागदपत्रांमध्ये किंवा कोडबेसमध्ये दिसू शकतात.
:::

ईव्हीएम कॉन्ट्रॅक्ट कॉल चॅलेंज ही स्पॅमविरोधी यंत्रणा आहे जी प्रकाशनाला परवानगी देण्यापूर्वी ऑन-चेन परिस्थितीची पडताळणी करते. मूळतः `plebbit-js` मधून स्वतंत्र पॅकेज म्हणून काढलेले, ते समुदाय मालकांना लेखकांना स्मार्ट-करार-परिभाषित निकषांची पूर्तता करणे आवश्यक आहे -- उदाहरणार्थ, किमान टोकन शिल्लक धारण -- पोस्ट करण्यासाठी.

**स्रोत कोड:** [github.com/bitsocialnet/evm-contract-call](https://github.com/bitsocialnet/evm-contract-call)

## आवश्यकता

- **Node.js** >= 22
- **ईएसएम-केवळ** -- हे पॅकेज CommonJS बिल्ड्स पाठवत नाही.
- **रनटाइम पीअर अवलंबित्व:** `@plebbit/plebbit-js` (`@pkc/pkc-js` वर स्थलांतरित करणे)

## स्थापना

```bash
npm install @bitsocial/evm-contract-challenge
```

## कॉन्फिगरेशन पर्याय

| पर्याय      | प्रकार   | वर्णन                                                                                      |
| ----------- | -------- | ------------------------------------------------------------------------------------------ |
| `matic`     |
| `string`    |
| `abi`       | `string` | कॉल केलेल्या कार्यासाठी ABI खंड.                                                           |
| `condition` | `string` | कॉन्ट्रॅक्ट रिटर्न व्हॅल्यू (उदा. `> 1000`) विरुद्ध मूल्यमापन केलेली तुलनात्मक अभिव्यक्ती. |
| `error`     | `string` | अट पूर्ण न करणाऱ्या लेखकांना दाखवलेला त्रुटी संदेश.                                        |

## उदाहरण

विशिष्ट ERC-20 टोकन असलेल्या 1,000 पेक्षा जास्त लेखकांना पोस्ट करणे प्रतिबंधित करू इच्छिणारा समुदाय मालक यासह आव्हान कॉन्फिगर करेल:

- `chainTicker`: `"eth"`
- `address`: टोकन कराराचा पत्ता
- `abi`: `balanceOf(address)` साठी ABI
- `condition`: `"> 1000"`
- `error`: `"You must hold more than 1,000 tokens to post in this community."`

लेखकाला कॉल प्रकाशित करण्याचा प्रयत्न करा लेखकाच्या पत्त्यासह `balanceOf` आणि परत केलेले मूल्य अट पूर्ण करते की नाही ते तपासते. तसे झाल्यास, प्रकाशन पुढे जाईल; अन्यथा, कॉन्फिगर केलेला एरर मेसेज परत केला जातो.

## ते कधी वापरायचे

EVM कॉन्ट्रॅक्ट कॉल चॅलेंज यासाठी आदर्श आहे:

- **टोकन-गेट केलेले समुदाय** जे टोकन धारकांना पोस्ट करणे प्रतिबंधित करतात.
- \*\* NFT-गेटेड ॲक्सेसची विशिष्ट मालकी असते. आवश्यक.
- **DAO गव्हर्नन्स स्पेस** जिथे सहभाग केवळ गव्हर्नन्स टोकन धारकांपुरता मर्यादित आहे.

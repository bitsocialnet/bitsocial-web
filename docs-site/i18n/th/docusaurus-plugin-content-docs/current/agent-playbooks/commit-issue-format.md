# รูปแบบความมุ่งมั่นและปัญหา

ใช้สิ่งนี้เมื่อนำเสนอหรือดำเนินการเปลี่ยนแปลงโค้ดที่มีความหมาย

## แบบฟอร์มข้อเสนอแนะความมุ่งมั่น

- **หัวข้อ:** รูปแบบ Conventional Commits สั้น ล้อมรอบด้วย backticks
- ใช้ `perf` (ไม่ใช่ `fix`) เพื่อเพิ่มประสิทธิภาพการทำงาน
- **คำอธิบาย:** ตัวเลือก 2-3 ประโยคที่ไม่เป็นทางการซึ่งอธิบายวิธีแก้ปัญหา กระชับ เทคนิค ไม่มีหัวข้อย่อย

ตัวอย่าง:

> **ชื่อคอมมิต:** `fix: correct date formatting in timezone conversion`
>
> อัปเดต `formatDate()` ใน `date-utils.ts` เพื่อจัดการการชดเชยเขตเวลาอย่างถูกต้อง

## รูปแบบข้อเสนอแนะปัญหา GitHub

- **หัวข้อ:** สั้นที่สุดเท่าที่จะทำได้ ล้อมรอบด้วย backticks
- **คำอธิบาย:** ประโยคที่ไม่เป็นทางการ 2-3 ประโยคที่อธิบายปัญหา (ไม่ใช่วิธีแก้ปัญหา) ราวกับว่ายังไม่ได้รับการแก้ไข

ตัวอย่าง:

> **ปัญหา GitHub:**
>
> - **หัวข้อ:** `Date formatting displays incorrect timezone`
> - **คำอธิบาย:** การประทับเวลาของความคิดเห็นแสดงเขตเวลาที่ไม่ถูกต้องเมื่อผู้ใช้ดูโพสต์จากภูมิภาคต่างๆ ฟังก์ชัน `formatDate()` ไม่ได้คำนึงถึงการตั้งค่าเขตเวลาท้องถิ่นของผู้ใช้

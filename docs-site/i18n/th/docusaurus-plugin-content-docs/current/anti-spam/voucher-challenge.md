# ความท้าทายบัตรกำนัล

Voucher Challenge เป็นกลไกป้องกันสแปมที่ควบคุมการเผยแพร่เนื้อหาโดยใช้รหัสบัตรกำนัลที่ไม่ซ้ำใคร แทนที่จะอาศัยการตรวจจับอัตโนมัติ ระบบจะเปลี่ยนความไว้วางใจไปที่เจ้าของชุมชนซึ่งจะแจกจ่ายรหัสให้กับบุคคลที่พวกเขาไว้วางใจด้วยตนเอง

**ซอร์สโค้ด:** [github.com/bitsocialnet/voucher-challenge](https://github.com/bitsocialnet/voucher-challenge)

## มันทำงานอย่างไร

1. เจ้าของชุมชนสร้างรหัสบัตรกำนัลที่ไม่ซ้ำกันตั้งแต่หนึ่งรหัสขึ้นไป
2. เจ้าของแจกจ่ายรหัสเหล่านั้นให้กับผู้เขียนที่เชื่อถือได้ผ่านช่องทางที่พวกเขาเลือก (ข้อความโดยตรง อีเมล ด้วยตนเอง ฯลฯ)
3. เมื่อผู้เขียนพยายามเผยแพร่ ระบบท้าทายจะแจ้งให้ป้อนรหัสบัตรกำนัล
4. รหัสได้รับการตรวจสอบแล้ว หากเป็นของแท้และยังไม่ได้ใช้ สิ่งพิมพ์จะได้รับการยอมรับ

รหัสบัตรกำนัลแต่ละรหัสจะเชื่อมโยงกับผู้เขียนเฉพาะเมื่อแลกแล้ว เพื่อป้องกันไม่ให้ผู้อื่นนำมาใช้ซ้ำ

## เมื่อใดควรใช้มัน

Voucher Challenge เหมาะที่สุดสำหรับ:

- **ชุมชนที่ได้รับเชิญเท่านั้น** ในกรณีที่สมาชิกถูกจำกัดโดยเจตนา
- **พื้นที่ที่ได้รับการดูแลจัดการ** ซึ่งเจ้าของจะคอยดูแลผู้เข้าร่วมทุกคนเป็นการส่วนตัว
- **สภาพแวดล้อมที่มีความน่าเชื่อถือสูง** ซึ่งการให้คะแนนสแปมอัตโนมัติไม่จำเป็นหรือไม่พึงประสงค์

เนื่องจากต้องใช้การแจกจ่ายโค้ดด้วยตนเอง จึงไม่ขยายขนาดไปยังชุมชนเปิดขนาดใหญ่ สำหรับสถานการณ์เหล่านั้น ให้พิจารณา [ตัวบล็อกของเรา](./spam-blocker.md) หรือ [ รูปแบบในการเรียกสัญญา EVM](./evm-contract-call.md) แทน

## บูรณาการ

Voucher Challenge เสียบเข้ากับอินเทอร์เฟซการท้าทายเดียวกันกับที่ใช้โดยแพ็คเกจป้องกันสแปมอื่น ๆ ในระบบนิเวศของ Bitsocial เจ้าของชุมชนเปิดใช้งานผ่านการตั้งค่าชุมชนของตน และความท้าทายจะถูกนำเสนอต่อผู้เขียนโดยอัตโนมัติเมื่อพวกเขาพยายามโพสต์

# Формат фіксації та випуску

Використовуйте це, коли пропонуєте або впроваджуєте значні зміни коду.

## Формат пропозиції щодо фіксації

- **Назва:** Стиль звичайних комітів, короткий, загорнутий у зворотні галочки.
- Використовуйте `perf` (а не `fix`) для оптимізації продуктивності.
- **Опис:** Необов’язкові 2-3 неофіційні речення з описом рішення. Лаконічно, технічно, без пунктів.

приклад:

> **Назва коміту:** `fix: correct date formatting in timezone conversion`
>
> Оновлено `formatDate()` у `date-utils.ts` для належної обробки зсувів часового поясу.

## Формат пропозиції щодо проблеми GitHub

- **Назва:** Якомога коротша, загорнута в зворотні галочки.
- **Опис:** 2-3 неофіційних речення, що описують проблему (а не рішення), наче все ще не вирішену.

приклад:

> **Проблема GitHub:**
>
> - **Назва:** `Date formatting displays incorrect timezone`
> - **Опис:** часові позначки коментарів показують неправильні часові пояси, коли користувачі переглядають публікації з різних регіонів. Функція `formatDate()` не враховує налаштування локального часового поясу користувача.

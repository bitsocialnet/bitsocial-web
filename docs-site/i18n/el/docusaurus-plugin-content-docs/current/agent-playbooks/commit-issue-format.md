# Μορφή δέσμευσης και έκδοσης

Χρησιμοποιήστε το όταν προτείνετε ή εφαρμόζετε σημαντικές αλλαγές κώδικα.

## Μορφή πρότασης δέσμευσης

- **Τίτλος:** Συμβατικό στιλ Commits, κοντό, τυλιγμένο σε backticks.
- Χρησιμοποιήστε το `perf` (όχι το `fix`) για βελτιστοποιήσεις απόδοσης.
- **Περιγραφή:** Προαιρετικές 2-3 άτυπες προτάσεις που περιγράφουν τη λύση. Συνοπτικό, τεχνικό, χωρίς κουκκίδες.

Παράδειγμα:

> **Τίτλος δέσμευσης:** `fix: correct date formatting in timezone conversion`
>
> Ενημερώθηκε το `formatDate()` στο `date-utils.ts` για τη σωστή διαχείριση των μετατοπίσεων ζώνης ώρας.

## Μορφή πρότασης ζητήματος GitHub

- **Τίτλος:** Όσο το δυνατόν πιο σύντομος, τυλιγμένος σε μπαστούνια.
- **Περιγραφή:** 2-3 άτυπες προτάσεις που περιγράφουν το πρόβλημα (όχι τη λύση), σαν να μην έχουν επιλυθεί ακόμα.

Παράδειγμα:

> **Πρόβλημα GitHub:**
>
> - **Τίτλος:** `Date formatting displays incorrect timezone`
> - **Περιγραφή:** Οι χρονικές σημάνσεις σχολίων εμφανίζουν λανθασμένες ζώνες ώρας όταν οι χρήστες προβάλλουν αναρτήσεις από διαφορετικές περιοχές. Η συνάρτηση `formatDate()` δεν λαμβάνει υπόψη τις ρυθμίσεις τοπικής ζώνης ώρας του χρήστη.

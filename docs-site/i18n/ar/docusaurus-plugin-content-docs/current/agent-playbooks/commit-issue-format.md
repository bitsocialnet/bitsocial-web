# الالتزام وشكل الإصدار

استخدم هذا عند اقتراح أو تنفيذ تغييرات ذات معنى في التعليمات البرمجية.

## الالتزام بتنسيق الاقتراح

- **العنوان:** أسلوب الالتزامات التقليدي، قصير، ملفوف بعلامات خلفية.
- استخدم `perf` (وليس `fix`) لتحسين الأداء.
- **الوصف:** اختياري 2-3 جمل غير رسمية تصف الحل. موجزة وتقنية ولا تحتوي على نقاط.

مثال:

> **عنوان الالتزام:** `fix: correct date formatting in timezone conversion`
>
> تم تحديث `formatDate()` في `date-utils.ts` للتعامل مع إزاحات المنطقة الزمنية بشكل صحيح.

## تنسيق اقتراح مشكلة GitHub

- **العنوان:** قصير قدر الإمكان، ملفوف في علامات خلفية.
- **الوصف:** 2-3 جمل غير رسمية تصف المشكلة (وليس الحل)، كما لو أنها لم يتم حلها بعد.

مثال:

> **مشكلة جيثب:**
>
> - **العنوان:** `Date formatting displays incorrect timezone`
> - **الوصف:** تُظهر الطوابع الزمنية للتعليقات مناطق زمنية غير صحيحة عندما يعرض المستخدمون منشورات من مناطق مختلفة. لا تأخذ وظيفة `formatDate()` في الاعتبار إعدادات المنطقة الزمنية المحلية للمستخدم.

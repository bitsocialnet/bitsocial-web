# 바우처 챌린지

바우처 챌린지는 고유한 바우처 코드 뒤에 콘텐츠 게시를 제어하는 ​​스팸 방지 메커니즘입니다. 자동화된 감지에 의존하는 대신, 신뢰하는 사람에게 코드를 수동으로 배포하는 커뮤니티 소유자에게 신뢰가 이전됩니다.

**소스 코드:** [github.com/bitsocialnet/voucher-challenge](https://github.com/bitsocialnet/voucher-challenge)

## 작동 방식

1. 커뮤니티 소유자는 하나 이상의 고유 바우처 코드를 생성합니다.
2. 소유자는 해당 코드를 자신이 선택한 채널을 통해 신뢰할 수 있는 작성자에게 배포합니다(직접 메시지, 이메일, 직접 방문 등).
3. 저자가 출판을 시도하면 챌린지 시스템에서 바우처 코드를 입력하라는 메시지를 표시합니다.
4. 코드가 검증됩니다. 코드가 진짜이고 아직 사용되지 않은 경우 출판이 허용됩니다.

각 바우처 코드는 일단 사용되면 특정 작성자와 연결되어 다른 사람이 재사용할 수 없도록 합니다.

## 사용 시기

바우처 챌린지가 가장 적합합니다. 대상:

- **회원 자격이 의도적으로 제한된 초대 전용 커뮤니티**.
- **관리된 공간** 소유자가 모든 참가자를 개인적으로 조사합니다.
- **자동 스팸 점수가 불필요하거나 바람직하지 않은 신뢰도가 높은 환경**.

수동 코드 배포가 필요하므로 대규모 공개 커뮤니티로 확장할 수 없습니다. 이러한 시나리오의 경우 [스팸]을 고려하세요. 대신 차단기](./spam-blocker.md) 또는 [EVM 호출 챌린지](./evm-contract-call.md).

## 통합

바우처 챌린지는 커뮤니티 소유자가 커뮤니티 설정을 통해 이를 활성화하고 게시를 시도할 때 작성자에게 자동으로 챌린지를 표시합니다.

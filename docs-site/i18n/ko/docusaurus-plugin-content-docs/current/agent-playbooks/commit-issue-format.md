# 커밋 및 문제 형식

의미 있는 코드 변경을 제안하거나 구현할 때 이 형식을 사용하세요.

## 커밋 제안 형식

- **제목:** 기존 커밋 스타일, 짧게, 백틱으로 묶음.
- 성능을 위해 `perf`(`fix` 아님)를 사용하세요. 최적화.
- **설명:** 솔루션을 설명하는 비공식 문장 2~3개(선택 사항). 간결하고 기술적이며 글머리 기호가 없습니다.

예:

> **커밋 제목:** `fix: correct date formatting in timezone conversion`
>
> 시간대 오프셋을 적절하게 처리하기 위해 `date-utils.ts`에서 `formatDate()`를 업데이트했습니다.

## GitHub 문제 제안 형식

- **제목:** 가능한 한 짧게, 백틱으로 묶음.
- **설명:** 마치 아직 해결되지 않은 것처럼 문제(해결책 아님)를 설명하는 2-3개의 비공식 문장.

예:

> **GitHub 문제:**
>
> - **제목:** `Date formatting displays incorrect timezone`
> - **설명:** 사용자가 다른 지역의 게시물을 볼 때 댓글 타임스탬프에 잘못된 시간대가 표시됩니다. `formatDate()` 함수는 사용자의 현지 시간대 설정을 고려하지 않습니다.

# アイデンティティとコミュニティの所有権

Bitsocial のプロファイルとコミュニティは、取り消し可能なプラットフォーム アカウントではなく、キーペアによって制御されます。

これにより、所有権とサービス プロバイダーの関係が変わります。サービスに ID やコミュニティ自体を永続的に管理することなく、稼働時間、ホスティング、利便性を別のサービスに委任できます。

## ここでキーが重要である理由

ホストされるプラットフォーム アカウントは、最終的にはレンタルされます。プラットフォームがそれを尊重し続ける限り、それは存在します。

キーで管理される ID は異なります:

- 秘密キーが権限のソースである
- ホスティングは置き換え可能
- アプリの選択は変更可能
- 所有権はインフラストラクチャとともに移動する必要はありません

## 委任の意味

委任は支援を意味する必要があります制御の放棄ではなく、可用性です。

たとえば、パブリック RPC またはホスティング サービスはノードをオンラインに保つことができますが、そのサービスにはプロファイルやコミュニティを永続的に占有する権限が必要ではありません。援助は役に立ちます。管理は任意のままであるべきです。

## より広範な点

これにより、Bitsocial ID は、企業から借りたユーザー名よりも、ウォレットで管理される資産に近く感じられます。サービス層は交換可能です。所有者関係はその交換後も存続する必要があります。

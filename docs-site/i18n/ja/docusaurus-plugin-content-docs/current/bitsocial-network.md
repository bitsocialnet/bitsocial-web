# Bitsocial Network

Bitsocial Network は、Bitsocial アプリの下に位置する共有アプリチェーン レイヤーのマスター プランで使用される名前です。

このアイデアは、ピアツーピア ソーシャル レイヤーを置き換えるものではありません。これは、ソーシャル アプリが既存のプラットフォームと大規模に競合する場合に必要となる共有の経済的および調整の基本要素を提供することです。

## ZXQPLACEHOLDER0ZXQ などの分散型 Bitsocial ドメインを強化する意味

-
- アプリ間で流動性を共有
- 永続的な収益化レール
- 銀行やプラットフォームが阻止しにくい金融構造
- スタック全体を所有する1つの企業に依存しないネットワーク効果

## それが重要な理由

コミュニティとアイデンティティの分散化は必要だが、ソーシャルメディアのすべてを分散化するだけでは十分ではない。

経済層が依然として少数の集中化されたレールに依存している場合、アプリは依然として、圧力をかけられたり、プラットフォームから外されたり、財政的に困窮したりしやすいままです。 Bitsocial Network は、依存関係の 2 番目の層に対する提案された答えです。

## Bitsocial の残りの部分との関係

Bitsocial Network は、ユーザーが毎日直接やり取りする製品としてではなく、Bitsocial アプリの共有インフラストラクチャとして提案されています。

一般向けの結果は次のようになります。

- コミュニティはピアツーピアのまま
- アプリはそのまま差別化
- 収益化とネーミングがより永続的になる
- 一元化されたプラットフォーム所有者を再作成することなく、エコシステム全体で価値を移動できる

## これがフェーズ 4 にある理由

マスタープランでは、これを以前のアプリレベルの証明ポイントの後に配置します。

1. イメージボードを分散化する
2. Bitsocial を起動するRPC
3. フォーラムを分散化
4. Bitsocial Network を起動

実際のアプリと実際のユーザーの行動がネットワーク上にすでに存在すると、エコシステム レベルの調整レイヤーがより有用になるため、このシーケンスは理にかなっています。

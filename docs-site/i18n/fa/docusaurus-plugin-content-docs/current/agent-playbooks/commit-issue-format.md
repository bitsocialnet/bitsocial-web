# فرمت تعهد و صدور

هنگام پیشنهاد یا اجرای تغییرات معنی دار کد از این مورد استفاده کنید.

## فرمت پیشنهاد متعهد

- **عنوان:** سبک متعارف متعهد، کوتاه، پیچیده شده در بکتیک.
- برای بهینه سازی عملکرد از `perf` (نه `fix`) استفاده کنید.
- **توضیحات:** اختیاری 2-3 جمله غیر رسمی که راه حل را توصیف می کند. مختصر، فنی، بدون نکات مهم.

مثال:

> **عنوان تعهد:** `fix: correct date formatting in timezone conversion`
>
> `formatDate()` را در `date-utils.ts` به‌روزرسانی کرد تا به درستی جابجایی‌های منطقه زمانی را مدیریت کند.

## فرمت پیشنهاد مسئله GitHub

- **عنوان:** تا حد امکان کوتاه، در بکتیک پیچیده شده است.
- **توضیحات:** 2-3 جمله غیررسمی که مشکل را توصیف می کند (نه راه حل)، که انگار هنوز حل نشده است.

مثال:

> **مشکل GitHub:**
>
> - **عنوان:** `Date formatting displays incorrect timezone`
> - **توضیحات:** زمانی که کاربران پست‌های مناطق مختلف را مشاهده می‌کنند، مهرهای زمانی نظر، مناطق زمانی نادرست را نشان می‌دهند. تابع `formatDate()` تنظیمات منطقه زمانی محلی کاربر را در نظر نمی گیرد.

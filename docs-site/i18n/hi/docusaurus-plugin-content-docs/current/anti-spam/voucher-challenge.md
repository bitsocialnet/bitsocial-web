# वाउचर चुनौती

वाउचर चैलेंज एक एंटी-स्पैम तंत्र है जो अद्वितीय वाउचर कोड के पीछे सामग्री प्रकाशन को गेट करता है। स्वचालित पहचान पर भरोसा करने के बजाय, यह समुदाय के मालिक पर भरोसा स्थानांतरित करता है, जो उन लोगों को मैन्युअल रूप से कोड वितरित करता है जिन पर वे भरोसा करते हैं।

**स्रोत कोड:** [github.com/bitsocialnet/voucher-challenge](https://github.com/bitsocialnet/voucher-challenge)

## यह कैसे काम करता है

1. एक समुदाय स्वामी एक या अधिक अद्वितीय वाउचर कोड उत्पन्न करता है।
2. स्वामी उन कोडों को विश्वसनीय लेखकों को उनकी पसंद के चैनल (प्रत्यक्ष संदेश, ईमेल, व्यक्तिगत रूप से, आदि) के माध्यम से वितरित करता है।
3. जब कोई लेखक प्रकाशित करने का प्रयास करता है, तो चुनौती प्रणाली उन्हें वाउचर कोड के लिए संकेत देती है।
4. कोड मान्य है - यदि यह वास्तविक है और पहले से उपयोग नहीं किया गया है, तो प्रकाशन स्वीकार कर लिया जाता है।

प्रत्येक वाउचर कोड एक बार भुनाए जाने के बाद एक विशिष्ट लेखक से जुड़ा होता है, जिससे दूसरों द्वारा पुन: उपयोग को रोका जा सकता है।

## इसका उपयोग कब करें

वाउचर चैलेंज इसके लिए सबसे उपयुक्त है:

- **केवल-आमंत्रित समुदाय** जहां सदस्यता जानबूझकर प्रतिबंधित है।
- **क्यूरेटेड स्थान** जहां मालिक व्यक्तिगत रूप से प्रत्येक प्रतिभागी की जांच करता है।
- **उच्च-भरोसेमंद वातावरण** जहां स्वचालित स्पैम स्कोरिंग अनावश्यक या अवांछनीय है।

क्योंकि इसमें मैन्युअल कोड वितरण की आवश्यकता होती है, यह बड़े खुले समुदायों तक सीमित नहीं है। उन परिदृश्यों के लिए, इसके बजाय [स्पैम ब्लॉक](./spam-blocker.md) या [ईएम अनुबंध कॉल चैलेंज](./evm-contract-call.md) पर विचार करें।

## एकीकरण

वाउचर चैलेंज बिटसोशल इकोसिस्टम में अन्य एंटी-स्पैम पैकेजों द्वारा उपयोग किए जाने वाले उसी चैलेंज इंटरफ़ेस में प्लग होता है। समुदाय के मालिक इसे अपनी समुदाय सेटिंग्स के माध्यम से सक्षम करते हैं, और जब लेखक पोस्ट करने का प्रयास करते हैं तो चुनौती स्वचालित रूप से प्रस्तुत की जाती है।

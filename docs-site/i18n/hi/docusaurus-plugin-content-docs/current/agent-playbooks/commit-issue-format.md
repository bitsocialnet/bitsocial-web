# प्रतिबद्धता और जारी करने का प्रारूप

सार्थक कोड परिवर्तनों का प्रस्ताव या कार्यान्वयन करते समय इसका उपयोग करें।

## सुझाव प्रारूप प्रस्तुत करें

- **शीर्षक:** पारंपरिक कमिट शैली, लघु, बैकटिक्स में लिपटी हुई।
- प्रदर्शन अनुकूलन के लिए `perf` (`fix` नहीं) का उपयोग करें।
- **विवरण:** समाधान का वर्णन करने वाले वैकल्पिक 2-3 अनौपचारिक वाक्य। संक्षिप्त, तकनीकी, कोई बुलेट पॉइंट नहीं।

उदाहरण:

> **प्रतिबद्ध शीर्षक:** `fix: correct date formatting in timezone conversion`
>
> समयक्षेत्र ऑफसेट को ठीक से संभालने के लिए `date-utils.ts` में `formatDate()` को अपडेट किया गया।

## GitHub अंक सुझाव प्रारूप

- **शीर्षक:** यथासंभव संक्षिप्त, बैकटिक्स में लपेटा हुआ।
- **विवरण:** 2-3 अनौपचारिक वाक्य जो समस्या का वर्णन करते हैं (समाधान नहीं), मानो अभी भी अनसुलझा हो।

उदाहरण:

> **गिटहब मुद्दा:**
>
> - **शीर्षक:** `Date formatting displays incorrect timezone`
> - **विवरण:** जब उपयोगकर्ता विभिन्न क्षेत्रों से पोस्ट देखते हैं तो टिप्पणी टाइमस्टैम्प गलत समय क्षेत्र दिखाते हैं। `formatDate()` फ़ंक्शन उपयोगकर्ता की स्थानीय समयक्षेत्र सेटिंग्स का ध्यान नहीं रखता है।

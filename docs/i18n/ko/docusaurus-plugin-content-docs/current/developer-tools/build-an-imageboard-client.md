# 이미지보드 클라이언트 빌드

1단계는 전체 카테고리를 포괄하는 하나의 공식 앱이 아닙니다. 5chan은 첫 번째 증거 포인트이지만 실제 목표는 다양한 시각적 언어, 조정 기본값, 디렉터리 모델 및 대상 커뮤니티를 갖춘 여러 Bitsocial 클라이언트 등 광범위한 이미지보드 생태계입니다.

## 1단계에 필요한 것

- 주류 온보딩을 위한 친숙한 4chan 스타일 클라이언트
- 다양한 문화와 보드 번들을 갖춘 Altchan에서 영감을 받은 고객
- 모바일 우선 또는 낮은 대역폭 클라이언트
- 확고한 자기주장을 갖고 있는 단일 커뮤니티 또는 틈새 클라이언트
- 첫 번째 앱과 함께 제공되는 것보다 더 나은 조정, 미디어, 온보딩 또는 검색 흐름

## 도움을 드리는 가장 빠른 방법

가장 빠른 배송 경로를 원하신다면 먼저 5chan에 직접 기여해 주세요:

- [5chan.app](https://5chan.app))에서 라이브 앱을 탐색해보세요
- [github.com/bitsocialnet/5chan](https://github.com/bitsocialnet/5chan)에서 소스를 검토하세요.
- [t.me/fivechandev](https://t.me/fivechandev)에서 빌더 채팅에 참여하세요.

## 나만의 클라이언트 구축

5chan이 원하는 커뮤니티나 인터페이스와 일치하지 않으면 대신 별도의 클라이언트를 구축하세요. 호환 가능한 Bitsocial 클라이언트는 동일한 제품 결정을 공유하지 않고도 동일한 네트워크를 공유할 수 있습니다.

1. 프로토콜 관련 도구를 알아보세요.
   - [비트소셜 리액트 후크](../react-hooks/)
   - [비트소셜 CLI](../cli/)
2. 실제로 어떤 종류의 이미지보드를 만들고 있는지 결정하세요.
보드 구조, 신원 가정, 조정 모델, 검색 흐름 및 시각적 언어를 먼저 선택하세요.
3. 프로젝트에 맞는 구현 경로를 선택하세요.
익숙한 이미지보드 베이스로 빠르게 움직이고 싶다면 5chan을 포크해보세요. UI 또는 상호 작용 모델이 근본적으로 달라야 하는 경우 새로 시작하세요.
4. 좁은 첫 번째 버전을 출시하세요.
하나의 실제 커뮤니티에 잘 서비스를 제공하는 클라이언트는 모든 사람을 만족시키려는 모호한 복제품보다 더 가치가 있습니다.
5. 결과를 게시하고 커뮤니티에서 테스트하도록 하세요.
Bitsocial은 하나의 공식 앱이 모든 작업을 수행할 때까지 기다리지 않고 외부 빌더가 제품 품질을 놓고 경쟁하는 독선적인 클라이언트를 출시할 때 개선됩니다.

## 디자인 원리

Bitsocial은 축복받은 고객이 한 명 있다고 해서 승리하는 것이 아닙니다. 많은 클라이언트가 공존하고, 포크하고, 전문화하고, 최초의 앱이 결코 할 수 없는 요구 사항을 충족할 수 있을 때 승리합니다.

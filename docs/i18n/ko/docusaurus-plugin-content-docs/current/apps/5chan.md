# 5chan

5chan은 서버가 없고 관리자가 없으며 Bitsocial 프로토콜에서 실행되는 완전히 분산된 이미지보드입니다. 이는 분산된 소유권을 도입하는 동시에 친숙한 이미지보드 디렉토리 구조를 따릅니다. 누구나 보드를 만들 수 있으며, 투표 메커니즘을 통해 여러 보드가 동일한 디렉토리 슬롯을 놓고 경쟁할 수 있습니다.

## 다운로드

| 플랫폼   | 링크                                |
| -------- | ----------------------------------- |
| 웹       | [5chan.app](https://5chan.app)      |
| 데스크탑 | Mac, Windows 및 Linux에서 사용 가능 |
| 모바일   | 안드로이드에서 사용 가능            |

## 보드 작동 방식

5chan은 클래식 디렉토리 레이아웃(예: `/b/`, `/g/`)을 사용하여 콘텐츠를 보드로 구성합니다. 중앙 관리자가 모든 보드를 제어하는 기존 이미지 보드와 달리 5chan에서는 모든 사용자가 자신의 보드를 만들고 완전히 소유할 수 있습니다. 여러 보드가 동일한 디렉터리 슬롯을 대상으로 하는 경우 투표를 통해 해당 위치를 놓고 경쟁합니다.

### 보드 만들기

새 보드를 생성하려면 `bitsocial-cli`를 P2P 노드로 실행해야 합니다. 이를 통해 중앙 서버에 의존하지 않고 보드가 분산된 방식으로 호스팅됩니다.

### 디렉터리 할당

디렉터리 슬롯 할당(어떤 보드가 어떤 경로에 나타나는지)은 현재 `5chan-directories.json` 파일에 대한 GitHub 풀 요청을 통해 관리됩니다. 이는 임시 프로세스입니다. 향후 릴리스에서는 인앱 보드 생성 및 pubsub 기반 투표를 지원하여 디렉토리 할당을 자동으로 처리할 것입니다.

## 내부

내부적으로 5chan은 네트워크 상호 작용을 위해 공유 Bitsocial 프로토콜 클라이언트 계층을 사용합니다. 5chan.app의 웹 앱은 고급 설정에서 브라우저 P2P가 활성화된 경우 브라우저에서 Helia 노드를 실행할 수도 있으므로 독자는 중앙 집중식 IPFS 게이트웨이 없이 피어에서 로드할 수 있습니다. P2P 프로토콜 참고 사항의 브라우저 P2P 섹션을 참조하세요.

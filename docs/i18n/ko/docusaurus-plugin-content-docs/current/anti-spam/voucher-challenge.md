# Voucher Challenge

바우처 챌린지는 고유한 바우처 코드 뒤에 콘텐츠 게시를 제어하는 ​​스팸 방지 메커니즘입니다. 자동화된 탐지에 의존하는 대신, 신뢰하는 사람들에게 코드를 수동으로 배포하는 커뮤니티 소유자에게 신뢰가 옮겨집니다.

**소스 코드:** [github.com/bitsocialnet/voucher-challenge](https://github.com/bitsocialnet/voucher-challenge)

## 작동 방식

1. 커뮤니티 소유자는 하나 이상의 고유 바우처 코드를 생성합니다.
2. 소유자는 자신이 선택한 채널(직접 메시지, 이메일, 직접 방문 등)을 통해 신뢰할 수 있는 작성자에게 해당 코드를 배포합니다.
3. 작성자가 출판을 시도하면 챌린지 시스템에서 바우처 코드를 입력하라는 메시지를 표시합니다.
4. 코드가 검증되었습니다. 코드가 정품이고 아직 사용되지 않은 경우 출판이 승인됩니다.

각 바우처 코드는 일단 사용되면 특정 작성자와 연결되므로 다른 사람이 재사용할 수 없습니다.

## 언제 사용해야 하는가

바우처 챌린지는 다음에 가장 적합합니다.

- **초대 전용 커뮤니티** 회원 자격이 의도적으로 제한되어 있습니다.
- 소유자가 모든 참가자를 직접 조사하는 **선별된 공간**.
- 자동화된 스팸 점수가 불필요하거나 바람직하지 않은 **신뢰도가 높은 환경**.

수동으로 코드를 배포해야 하기 때문에 대규모 개방형 커뮤니티로 확장되지 않습니다. 이러한 시나리오의 경우 대신 [Spam Blocker](./spam-blocker.md) 또는 [EVM Contract Call Challenge](./evm-contract-call.md)을 고려하세요.

## 완성

바우처 챌린지는 Bitsocial 생태계의 다른 스팸 방지 패키지에서 사용하는 것과 동일한 챌린지 인터페이스에 연결됩니다. 커뮤니티 소유자는 커뮤니티 설정을 통해 이 기능을 활성화하고 작성자가 게시를 시도할 때 작성자에게 자동으로 챌린지를 표시합니다.

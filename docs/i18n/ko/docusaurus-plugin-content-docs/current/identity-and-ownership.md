# 신원 및 커뮤니티 소유권

Bitsocial의 프로필과 커뮤니티는 취소 가능한 플랫폼 계정이 아닌 키 쌍으로 제어됩니다.

이는 소유권과 서비스 제공자 간의 관계를 변화시킵니다. 해당 서비스에 ID나 커뮤니티 자체에 대한 영구적인 관리권을 부여하지 않고도 가동 시간, 호스팅 또는 편의성을 다른 서비스에 위임할 수 있습니다.

## 여기서 키가 중요한 이유

호스팅 플랫폼 계정은 궁극적으로 임대됩니다. 플랫폼이 계속해서 이를 존중하는 한 존재합니다.

키 제어 ID는 다릅니다.

- 개인 키는 권한의 소스입니다
- 호스팅 대체 가능
- 앱 선택 변경 가능
- 소유권은 인프라와 함께 이동할 필요가 없습니다

## 위임의 의미

위임은 가용성에 대한 도움을 의미해야 합니다. 통제권을 포기하지 않습니다.

예를 들어, 공개 RPC 또는 호스팅 서비스는 노드를 온라인 상태로 유지할 수 있지만 해당 서비스는 프로필이나 커뮤니티를 영구적으로 점유할 수 있는 권한이 필요하지 않습니다. 도움이 유용합니다. 보관은 선택 사항으로 남아 있어야 합니다.

## 더 넓은 의미

이는 Bitsocial ID를 회사에서 임대한 사용자 이름보다 지갑이 관리하는 자산에 더 가깝게 느끼게 합니다. 서비스 계층을 교체할 수 있습니다. 소유자 관계는 해당 교환에서 살아남아야 합니다.

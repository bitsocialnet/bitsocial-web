# দীর্ঘ-চলমান এজেন্ট ওয়ার্কফ্লো

এই প্লেবুকটি ব্যবহার করুন যখন একটি টাস্ক একাধিক সেশন, হ্যান্ডঅফ বা স্পন এজেন্ট স্প্যান করার সম্ভাবনা থাকে।

## গোল

- প্রসঙ্গ পুনরুদ্ধার করার জন্য প্রতিটি নতুন সেশনকে একটি দ্রুত উপায় দিন
- বৃহৎ পরিবর্তনকে এক-শট করার পরিবর্তে ক্রমবর্ধমান রাখুন
- আরও কোড যোগ করার আগে একটি ভাঙা স্থানীয় বেসলাইন ধরুন
- টেকসই আর্টিফ্যাক্টগুলি ছেড়ে দিন যা পরবর্তী সেশন বিশ্বাস করতে পারে

## রাষ্ট্র কোথায় রাখবেন

- মানুষ, রিভিউ বট বা একাধিক টুলচেইনের একই টাস্ক স্টেটের প্রয়োজন হলে `docs/agent-runs/<slug>/` ব্যবহার করুন।
- একটি টুল-স্থানীয় ডিরেক্টরি ব্যবহার করুন যেমন `.codex/runs/<slug>/` শুধুমাত্র যখন টাস্ক স্টেট ইচ্ছাকৃতভাবে একটি ওয়ার্কস্টেশন বা একটি টুলচেইনের জন্য স্থানীয় হয়।
- কোনো ব্যক্তিগত স্ক্র্যাচ ফাইলে মাল্টি-সেশন শেয়ার্ড স্টেট লুকাবেন না যদি অন্য কোনো অবদানকারী বা এজেন্টের প্রয়োজন পরে।

## প্রয়োজনীয় ফাইল

দীর্ঘ-চলমান টাস্কের শুরুতে এই ফাইলগুলি তৈরি করুন:

- `feature-list.json`
- `progress.md`

`docs/agent-playbooks/templates/feature-list.template.json` এবং `docs/agent-playbooks/templates/progress.template.md`-এ টেমপ্লেটগুলি ব্যবহার করুন৷

বৈশিষ্ট্য তালিকার জন্য JSON পছন্দ করুন যাতে এজেন্টরা সম্পূর্ণ নথিটি পুনরায় লেখা ছাড়াই অল্প সংখ্যক ক্ষেত্র আপডেট করতে পারে।

## অধিবেশন শুরু চেকলিস্ট

1. `pwd` চালান।
2. `progress.md` পড়ুন।
3. `feature-list.json` পড়ুন।
4. `git log --oneline -20` চালান।
5. `./scripts/agent-init.sh --smoke` চালান।
6. ঠিক একটি সর্বোচ্চ অগ্রাধিকার আইটেম চয়ন করুন যা এখনও `pending`, `in_progress`, বা `blocked`৷

স্মোক স্টেপ ব্যর্থ হলে, একটি নতুন বৈশিষ্ট্য স্লাইস প্রয়োগ করার আগে ভাঙা বেসলাইন ঠিক করুন।

## সেশনের নিয়ম

- একবারে একটি বৈশিষ্ট্য বা টাস্ক স্লাইসে কাজ করুন।
- বৈশিষ্ট্য তালিকা মেশিন-পাঠযোগ্য এবং স্থিতিশীল রাখুন। সম্পর্কহীন আইটেমগুলি পুনরায় লেখার পরিবর্তে স্থিতি, নোট, ফাইল এবং যাচাইকরণ ক্ষেত্র আপডেট করুন।
- শুধুমাত্র সেই আইটেমে তালিকাভুক্ত কমান্ড বা ব্যবহারকারী প্রবাহ চালানোর পরে যাচাইকৃত একটি আইটেম চিহ্নিত করুন।
- বাউন্ডেড স্লাইসের জন্য স্পন এজেন্ট ব্যবহার করুন, সামগ্রিক টাস্ক-স্টেট মালিকানার জন্য নয়।
- যখন একটি শিশু এজেন্ট একটি আইটেমের মালিক হয়, তখন এটিকে সঠিক আইটেম আইডি, গ্রহণযোগ্যতার মানদণ্ড এবং ফাইলগুলি স্পর্শ করতে পারে।

## অধিবেশন শেষ চেকলিস্ট

1. `progress.md` এ একটি সংক্ষিপ্ত অগ্রগতি এন্ট্রি যোগ করুন।
2. `feature-list.json`-এ স্পর্শ করা আইটেম আপডেট করুন।
3. যাচাইয়ের জন্য চালানো সঠিক কমান্ডগুলি রেকর্ড করুন।
4. ক্যাপচার ব্লকার, ফলো-আপ, এবং পুনরায় শুরু করার জন্য পরবর্তী সেরা আইটেম।

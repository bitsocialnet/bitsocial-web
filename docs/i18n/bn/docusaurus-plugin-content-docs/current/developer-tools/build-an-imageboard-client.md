# একটি ইমেজবোর্ড ক্লায়েন্ট তৈরি করুন

ফেজ 1 পুরো বিভাগ কভার করে একটি অফিসিয়াল অ্যাপ সম্পর্কে নয়। 5chan হল প্রথম প্রুফ পয়েন্ট, কিন্তু আসল লক্ষ্য হল একটি বিস্তৃত ইমেজবোর্ড ইকোসিস্টেম: বিভিন্ন ভিজ্যুয়াল ল্যাঙ্গুয়েজ, মডারেশন ডিফল্ট, ডিরেক্টরি মডেল এবং টার্গেট সম্প্রদায় সহ একাধিক বিটসোশ্যাল ক্লায়েন্ট।

## কি ফেজ 1 প্রয়োজন

- মূলধারার অনবোর্ডিংয়ের জন্য পরিচিত 4chan-স্টাইলের ক্লায়েন্ট
- বিভিন্ন সংস্কৃতি এবং বোর্ড বান্ডিল সহ Altchan-অনুপ্রাণিত ক্লায়েন্ট
- মোবাইল-প্রথম বা কম ব্যান্ডউইথ ক্লায়েন্ট
- দৃঢ় মতামতযুক্ত ডিফল্ট সহ একক-সম্প্রদায় বা বিশেষ ক্লায়েন্ট
- প্রথম অ্যাপ পাঠানোর চেয়ে ভাল সংযম, মিডিয়া, অনবোর্ডিং বা আবিষ্কারের প্রবাহ

## সাহায্য করার দ্রুততম উপায়

আপনি যদি শিপিংয়ের সবচেয়ে সংক্ষিপ্ত পথ চান, তাহলে প্রথমে সরাসরি 5chan-এ অবদান রাখুন:

- [5chan.app](https://5chan.app).) এ লাইভ অ্যাপটি দেখুন
- [github.com/bitsocialnet/5chan](https://github.com/bitsocialnet/5chan)) এ উত্সটি পর্যালোচনা করুন
- বিল্ডার চ্যাটে যোগ দিন [t.me/fivechandev](https://t.me/fivechandev)

## আপনার নিজস্ব ক্লায়েন্ট তৈরি করুন

যদি 5chan আপনার পছন্দের সম্প্রদায় বা ইন্টারফেসের সাথে মেলে না, তবে পরিবর্তে একটি পৃথক ক্লায়েন্ট তৈরি করুন। সামঞ্জস্যপূর্ণ বিটসোশ্যাল ক্লায়েন্টরা একই পণ্যের সিদ্ধান্তগুলি ভাগ না করে একই নেটওয়ার্ক ভাগ করতে পারে।

1. প্রোটোকল-মুখী সরঞ্জামগুলি শিখুন:
   - [বিটসোশ্যাল রিঅ্যাক্ট হুক](../react-hooks/)
   - [বিটসোশ্যাল সিএলআই](../cli/)
2. আপনি আসলে কি ধরনের ইমেজবোর্ড তৈরি করছেন তা নির্ধারণ করুন।
প্রথমে বোর্ড গঠন, পরিচয় অনুমান, সংযম মডেল, আবিষ্কার প্রবাহ, এবং ভিজ্যুয়াল ভাষা চয়ন করুন।
3. প্রকল্পের সাথে মানানসই বাস্তবায়নের পথ বেছে নিন।
Fork 5chan যদি আপনি একটি পরিচিত ইমেজবোর্ড বেস দিয়ে দ্রুত সরতে চান। UI বা ইন্টারঅ্যাকশন মডেল আমূল ভিন্ন হওয়া প্রয়োজন হলে নতুন করে শুরু করুন।
4. একটি সংকীর্ণ প্রথম সংস্করণ জাহাজ.
একটি ক্লায়েন্ট যেটি একটি বাস্তব সম্প্রদায়কে ভালভাবে পরিবেশন করে তা সকলকে সন্তুষ্ট করার জন্য একটি অস্পষ্ট ক্লোনের চেয়ে বেশি মূল্যবান।
5. ফলাফল প্রকাশ করুন এবং সম্প্রদায়গুলিকে এটি পরীক্ষা করতে দিন।
বিটসোশ্যাল উন্নত হয় যখন বাইরের নির্মাতারা মতামতযুক্ত ক্লায়েন্ট পাঠায় যারা সবকিছু করার জন্য একটি অফিসিয়াল অ্যাপের জন্য অপেক্ষা করার পরিবর্তে পণ্যের গুণমান নিয়ে প্রতিযোগিতা করে।

## নকশা নীতি

Bitsocial একটি আশীর্বাদ ক্লায়েন্ট থাকার দ্বারা জয়ী হয় না. এটি জয়ী হয় যখন অনেক ক্লায়েন্ট সহাবস্থান করতে পারে, কাঁটাচামচ করতে পারে, বিশেষীকরণ করতে পারে, এবং পরিবেশন করতে পারে প্রথম অ্যাপটি কখনই হবে না।

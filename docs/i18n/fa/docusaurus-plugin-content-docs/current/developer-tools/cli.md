# CLI

:::warning نام‌گذاری قدیمی
این بسته در حال حاضر از قراردادهای نامگذاری قدیمی استفاده می کند که از وابستگی بالادستی آن به ارث رسیده است. ارجاعات به "plebbit" در دستورات، خروجی و پیکربندی در نسخه بعدی به "bitsocial" منتقل خواهند شد. عملکرد بی تاثیر است.
:::

`bitsocial-cli` یک ابزار خط فرمان برای تعامل با باطن پروتکل Bitsocial است. این به شما امکان می دهد یک شبح P2P محلی را اجرا کنید، انجمن ها را ایجاد و پیکربندی کنید، و محتوا را منتشر کنید - همه از ترمینال.

این بر روی `plebbit-js` ساخته شده است و توسط [5chan](/apps/5chan/) و [Seedit](/apps/seedit/) برای ایجاد انجمن و مدیریت گره استفاده می‌شود.

## نصب و راه اندازی

باینری های از پیش ساخته شده برای ویندوز، macOS و لینوکس در دسترس هستند. آخرین نسخه را برای پلتفرم خود از GitHub دانلود کنید:

**[دانلود از نسخه‌های GitHub](https://github.com/bitsocialnet/bitsocial-cli/releases)**

پس از دانلود، باینری را قابل اجرا (macOS/Linux) کنید:

```bash
chmod +x bitsocial-cli
```

## اجرای دیمون

رایج ترین استفاده از CLI اجرای یک گره Bitsocial است. دیمون لایه شبکه P2P را راه اندازی می کند و یک API محلی را نشان می دهد که مشتریان می توانند به آن متصل شوند.

```bash
bitsocial-cli daemon
```

در اولین راه‌اندازی، دیمون پیوندهایی را به **WebUI**، یک رابط گرافیکی مبتنی بر مرورگر برای مدیریت گره، جوامع و تنظیمات شما خروجی می‌دهد. اگر یک رابط کاربری گرافیکی را به دستورات ترمینال ترجیح می دهید، مفید است.

## دستورات کلیدی

| فرمان               | توضیحات                                             |
| ------------------- | --------------------------------------------------- |
| `daemon`            | گره Bitsocial P2P را راه اندازی کنید                |
| `create subplebbit` | ایجاد یک انجمن جدید                                 |
| `subplebbit edit`   | به روز رسانی تنظیمات انجمن (عنوان، توضیحات، قوانین) |
| `subplebbit list`   | فهرست انجمن های میزبانی شده در این گره              |
| `subplebbit start`  | شروع خدمت به یک جامعه خاص                           |
| `subplebbit stop`   | توقف خدمت به یک جامعه خاص                           |

هر دستوری را با `--help` اجرا کنید تا گزینه ها و پرچم های موجود را ببینید:

```bash
bitsocial-cli daemon --help
bitsocial-cli create subplebbit --help
```

## گردش کار معمولی

یک جریان راه اندازی رایج برای میزبانی یک جامعه جدید:

```bash
# 1. دیمون را شروع کنید
bitsocial-cli daemon

# 2. در یک ترمینال دیگر، یک جامعه ایجاد کنید
bitsocial-cli create subplebbit

# 3. جامعه را پیکربندی کنید
bitsocial-cli subplebbit edit <address> --title "My Community" --description "A decentralized forum"

# 4. شروع به سرو کردن کنید
bitsocial-cli subplebbit start <address>
```

انجمن اکنون در شبکه Bitsocial فعال است و از هر مشتری سازگار قابل دسترسی است.

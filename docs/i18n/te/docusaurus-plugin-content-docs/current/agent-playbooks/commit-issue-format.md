# కమిట్ మరియు ఇష్యూ ఫార్మాట్

అర్థవంతమైన కోడ్ మార్పులను ప్రతిపాదించేటప్పుడు లేదా అమలు చేస్తున్నప్పుడు దీన్ని ఉపయోగించండి.

## కమిట్ సూచన ఫార్మాట్

- **శీర్షిక:** సంప్రదాయ కమిట్‌ల శైలి, చిన్నది, బ్యాక్‌టిక్‌లతో చుట్టబడి ఉంటుంది.
- పనితీరు అనుకూలీకరణల కోసం `perf` (`fix` కాదు) ఉపయోగించండి.
- **వివరణ:** పరిష్కారాన్ని వివరించే ఐచ్ఛిక 2-3 అనధికారిక వాక్యాలు. సంక్షిప్త, సాంకేతిక, బుల్లెట్ పాయింట్లు లేవు.

ఉదాహరణ:

> **కమిట్ శీర్షిక:** `fix: correct date formatting in timezone conversion`
>
> టైమ్‌జోన్ ఆఫ్‌సెట్‌లను సరిగ్గా నిర్వహించడానికి `date-utils.ts`లో `formatDate()` నవీకరించబడింది.

## GitHub ఇష్యూ సూచన ఫార్మాట్

- **శీర్షిక:** వీలైనంత చిన్నది, బ్యాక్‌టిక్‌లతో చుట్టబడి ఉంటుంది.
- **వివరణ:** సమస్యను వివరించే 2-3 అనధికారిక వాక్యాలు (పరిష్కారం కాదు), ఇప్పటికీ పరిష్కరించబడలేదు.

ఉదాహరణ:

> **GitHub సమస్య:**
>
> - **శీర్షిక:** `Date formatting displays incorrect timezone`
> - **వివరణ:** వినియోగదారులు వివిధ ప్రాంతాల నుండి పోస్ట్‌లను వీక్షించినప్పుడు కామెంట్ టైమ్‌స్టాంప్‌లు తప్పు టైమ్‌జోన్‌లను చూపుతాయి. `formatDate()` ఫంక్షన్ వినియోగదారు యొక్క స్థానిక సమయమండలి సెట్టింగ్‌లకు కారణం కాదు.

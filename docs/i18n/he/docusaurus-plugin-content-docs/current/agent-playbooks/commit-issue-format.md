# התחייבות והנפקה פורמט

השתמש בזה בעת הצעה או יישום של שינויים משמעותיים בקוד.

## קבל פורמט הצעה

- **כותרת:** סגנון Commits קונבנציונלי, קצר, עטוף ב-backticks.
- השתמש ב-`perf` (לא ב-`fix`) עבור מיטוב ביצועים.
- **תיאור:** אופציונלי 2-3 משפטים לא רשמיים המתארים את הפתרון. תמציתי, טכני, ללא נקודות כדור.

דוגמה:

> **כותרת התחייבות:** `fix: correct date formatting in timezone conversion`
>
> עודכן את `formatDate()` ב-`date-utils.ts` כדי לטפל כראוי בקיזוז אזורי זמן.

## פורמט הצעה לבעיה של GitHub

- **כותרת:** קצרה ככל האפשר, עטופה ב-backticks.
- **תיאור:** 2-3 משפטים לא רשמיים המתארים את הבעיה (לא את הפתרון), כאילו עדיין לא נפתרה.

דוגמה:

> **בעיית GitHub:**
>
> - **כותרת:** `Date formatting displays incorrect timezone`
> - **תיאור:** חותמות זמן של הערות מציגות אזורי זמן שגויים כאשר משתמשים צופים בפוסטים מאזורים שונים. הפונקציה `formatDate()` אינה מתייחסת להגדרות אזור הזמן המקומי של המשתמש.

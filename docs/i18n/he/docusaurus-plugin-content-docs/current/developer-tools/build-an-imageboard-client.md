# בניית לקוח imageboard

שלב 1 אינו עוסק באפליקציה רשמית אחת המכסה את כל הקטגוריה. 5chan היא נקודת ההוכחה הראשונה, אך המטרה בפועל היא מערכת אקולוגית רחבה של לוח תמונות: מספר לקוחות Bitsocial עם שפות ויזואליות שונות, ברירות מחדל לניהול, מודלים של ספריות וקהילות יעד.

## מה שלב 1 צריך

- לקוחות מוכרים בסגנון 4chan לכניסה למיינסטרים
- לקוחות בהשראת Altchan עם תרבויות שונות וחבילות לוחות
- לקוחות ניידים או ברוחב פס נמוך
- לקוחות בני קהילה או נישה עם ברירות מחדל דעתניות חזקות
- זרימת ניהול, מדיה, כניסה או גילוי טובים יותר מאשר האפליקציה הראשונה שנשלחת איתה

## הדרך המהירה ביותר לעזור

אם אתה רוצה את הדרך הקצרה ביותר למשלוח, תרום תחילה ישירות ל-5chan:

- חקור את האפליקציה בשידור חי בכתובת [5chan.app](https://5chan.app)
- סקור את המקור בכתובת [github.com/bitsocialnet/5chan](https://github.com/bitsocialnet/5chan)
- הצטרף לצ'אט בונה בכתובת [t.me/fivechandev](https://t.me/fivechandev)

## בנה לקוח משלך

אם 5chan אינו תואם לקהילה או לממשק הרצוי, בנו במקום זאת לקוח נפרד. לקוחות Bitsocial תואמים יכולים לשתף את אותה רשת מבלי לשתף את אותן החלטות מוצר.

1. למד את הכלים הפונים לפרוטוקול:
   - [ווים של Bitsocial React](../react-hooks/)
   - [Bitsocial CLI](../cli/)
2. החלט איזה סוג של לוח תמונה אתה בעצם בונה.
תחילה בחרו את מבנה הלוח, הנחות הזהות, מודל המנחה, זרימת הגילוי והשפה החזותית.
3. בחר את נתיב היישום המתאים לפרויקט.
Fork 5chan אם אתה רוצה לנוע מהר עם בסיס imageboard מוכר. התחל מחדש אם ממשק המשתמש או מודל האינטראקציה צריכים להיות שונים בתכלית.
4. שלח גרסה ראשונה צרה.
לקוח אחד שמשרת היטב קהילה אמיתית אחת הוא בעל ערך רב יותר משכפול מעורפל שנועד לספק את כולם.
5. פרסם את התוצאה ואפשר לקהילות לבדוק אותה.
Bitsocial משתפר כאשר בונים חיצוניים שולחים לקוחות דעתניים שמתחרים על איכות המוצר במקום לחכות לאפליקציה רשמית אחת שתעשה הכל.

## עקרון עיצוב

Bitsocial לא זוכה בכך שיש לו לקוח מבורך אחד. זה מנצח כאשר לקוחות רבים יכולים להתקיים במקביל, להתפצל, להתמחות ולשרת צרכים שהאפליקציה הראשונה לעולם לא תעשה.

# Αποκέντρωση του Twitter/X

Η Φάση 3 είναι ένα σχέδιο για τη δημιουργία μιας εστιασμένης, αποκεντρωμένης εναλλακτικής του Twitter/X. Στο επίκεντρό της βρίσκεται η δημόσια συζήτηση που βασίζεται στο κείμενο: σύντομες αναρτήσεις, απαντήσεις, αναδημοσιεύσεις, ακολουθήσεις, συζήτηση σε πραγματικό χρόνο και κοινότητες, με το επίπεδο της πλατφόρμας ανοιχτό.

Το Twitter/X εξακολουθεί να ορίζεται από τις αναρτήσεις, το κείμενο και την ανταλλαγή ιδεών. Ο client της Φάσης 3 θα πρέπει να ανταγωνιστεί σε αυτή τη βασική εμπειρία και να την κάνει να λειτουργεί εξαιρετικά καλά.

Αυτή η σελίδα περιγράφει την κατεύθυνση του προϊόντος, όχι μια κλειδωμένη προδιαγραφή έκδοσης. Η ακριβής διεπαφή, η προεπιλεγμένη ροή, το μοντέλο διαφημίσεων, οι λειτουργίες AI και η αγορά RPC μπορούν να αλλάξουν καθώς ωριμάζουν το πρωτόκολλο και οι πρώτες εφαρμογές.

## Τι πρέπει να αποδείξει

Ο client θα πρέπει να αποδείξει ότι ένα κοινωνικό δίκτυο βασισμένο σε προφίλ μπορεί να αποφύγει να γίνει πλατφόρμα θεματοφυλακής:

- οι χρήστες μπορούν να κατέχουν τις ταυτότητες και τα προφίλ τους
- οι κοινότητες και οι κόμβοι προφίλ μπορούν να παραμείνουν peer-to-peer
- οι κοινότητες μπορούν να μεταφέρουν αποτελέσματα δικτύου μεταξύ των clients του Bitsocial
- οι πάροχοι RPC μπορούν να κάνουν τον client εύχρηστο χωρίς να αναλαμβάνουν τη θεματοφυλακή
- οι αλγόριθμοι ροής μπορούν να είναι προαιρετικές υπηρεσίες αντί για κανόνες που επιβάλλει η πλατφόρμα
- άλλοι clients μπορούν να συνεχίσουν να ανταγωνίζονται στο ίδιο δίκτυο

Ο στόχος είναι να δημιουργηθεί ο ισχυρότερος δυνατός client για δημόσια συζήτηση και να αποδειχθεί πόσο μακριά μπορεί να φτάσει το πρωτόκολλο.

## Οικείος στον σκοπό, αντικαταστάσιμος από σχεδιασμό

Η προεπιλεγμένη εμπειρία θα πρέπει να ανταγωνίζεται τον πυρήνα του Twitter/X: γρήγορη αρχική ροή, αναρτήσεις κειμένου, ακολουθήσεις, απαντήσεις, διάδοση μέσω αναδημοσιεύσεων, κοινότητες, ειδοποιήσεις, αναζήτηση και μια ταξινομημένη προβολή For You που λειτουργεί αμέσως.

Το Bitsocial Forge μπορεί να λειτουργήσει την πρώτη προεπιλεγμένη υπηρεσία RPC και ροής. Αυτή η προεπιλογή μπορεί να περιλαμβάνει ταξινομημένη ροή και διαφημίσεις, ώστε ο client να φαίνεται πλήρης από την πρώτη ημέρα αντί να ζητά από τους συνηθισμένους χρήστες να συναρμολογήσουν μόνοι τους ολόκληρο το σύστημα.

Η διαφορά είναι ότι η προεπιλογή δεν πρέπει να γίνει φυλακή. Ο χρήστης θα πρέπει να μπορεί να αλλάξει RPC, ροές, instances, συστήματα κατάταξης, διαφημίσεις και παρόχους ανακάλυψης ή να αφαιρέσει εντελώς την κατάταξη. Ο client μπορεί να κάνει σαφείς προεπιλεγμένες επιλογές κατά την πρώτη εκκίνηση, διατηρώντας παράλληλα κάθε σημαντική υπηρεσία αντικαταστάσιμη.

Έτσι ο client γίνεται πιο προσαρμόσιμος από μια συμβατική πλατφόρμα. Ένας χρήστης μπορεί να κρατήσει την προεπιλεγμένη ταξινομημένη ροή με διαφημίσεις. Ένας άλλος μπορεί να χρησιμοποιήσει χρονολογική ροή χωρίς κατάταξη. Ένας τρίτος μπορεί να επιλέξει ένα RPC που εστιάζει στο απόρρητο, μια υπηρεσία ανακάλυψης που διαχειρίζεται η κοινότητα, μια επί πληρωμή ροή χωρίς διαφημίσεις ή έναν εξειδικευμένο αλγόριθμο για μια συγκεκριμένη υποκουλτούρα.

## Κοινότητες μεταξύ clients

Οι κοινότητες θα πρέπει να είναι πολύ σημαντικότερες από απομονωμένες ομάδες μέσα σε μία εφαρμογή.

Στο X/Twitter, οι κοινότητες περιορίζονται μέσα στο X. Μπορούν να είναι χρήσιμες, αλλά κληρονομούν τους περιορισμούς μίας πλατφόρμας, ενός συστήματος λογαριασμών, μίας στοίβας προτάσεων και μίας επιφάνειας προϊόντος.

Μια κοινότητα Bitsocial μπορεί να δημιουργηθεί, να φιλοξενηθεί, να ανακαλυφθεί και να χρησιμοποιηθεί μέσω διαφορετικών clients. Αυτό σημαίνει ότι ο client της Φάσης 3 μπορεί να εμφανίζει κοινότητες και αναρτήσεις από το ευρύτερο δίκτυο Bitsocial, όχι μόνο από χρήστες που ξεκίνησαν μέσα σε αυτόν. Μια κοινότητα θα μπορούσε να έχει ταυτόχρονα δραστηριότητα από έναν client imageboard, έναν client συζητήσεων τύπου Reddit, έναν client εξειδικευμένου φόρουμ, μια εφαρμογή για κινητά και τον client της Φάσης 3.

Αυτό είναι το βασικό πλεονέκτημα του αποτελέσματος δικτύου: ένας client μπορεί να φαίνεται οικείος στους συνηθισμένους χρήστες και ταυτόχρονα να αντλεί αξία από πολλούς clients, κόμβους κοινοτήτων, παρόχους RPC και ανεξάρτητες υπηρεσίες.

## Προαιρετικοί αλγόριθμοι ροής

Ο client της Φάσης 3 δεν θα πρέπει να επιβάλλει σε όλους ένα ενιαίο παγκόσμιο σύστημα κατάταξης.

Οι αλγόριθμοι ροής θα πρέπει να ενεργοποιούνται κατ' επιλογή. Ο χρήστης θα μπορούσε να επιλέξει έναν αλγόριθμο από μια αγορά, να αλλάξει πάροχο, να χρησιμοποιήσει τον αλγόριθμο μιας εταιρείας, ενός ανώνυμου διαχειριστή ή μιας κοινότητας, να εκτελέσει έναν προσωπικό αλγόριθμο ή να μη χρησιμοποιήσει κανέναν.

Οι δημόσιοι πάροχοι RPC αποτελούν φυσικό χώρο ανταγωνισμού για αυτές τις υπηρεσίες. Μπορούν να ευρετηριάζουν, να κατατάσσουν και να προτείνουν περιεχόμενο, αλλά δεν θα πρέπει να κατέχουν τον χρήστη ή το προφίλ.

Αυτές οι υπηρεσίες μπορούν επίσης να ανταγωνίζονται ως προς τη μορφή του ίδιου του client. Ένα RPC μπορεί να παρέχει ταξινομημένη ροή με διαφημίσεις. Ένα άλλο μπορεί να παρέχει αταξινόμητη χρονολογική ροή. Ένα τρίτο μπορεί να ειδικεύεται στο απόρρητο, τη μετάφραση, τον συντονισμό, την ανακάλυψη κοινοτήτων ή σε ένα εξειδικευμένο κοινωνικό γράφημα.

Αν το οικονομικό μοντέλο λειτουργήσει, οι υπηρεσίες ροής που βασίζονται σε RPC θα μπορούσαν να προσθέσουν λειτουργίες AI παρόμοιες με εκείνες που οι δημοφιλείς πλατφόρμες προσπαθούν να ενσωματώσουν στις ροές τους: αυτόματες μεταφράσεις, περιλήψεις, απαντήσεις με τη βοήθεια bot, απαντήσεις αναζήτησης, βοήθεια στον συντονισμό ή περιεχόμενο τύπου σημειώσεων κοινότητας.

Αυτές οι λειτουργίες θα πρέπει να είναι επιλογές υπηρεσίας, όχι απαιτήσεις του πρωτοκόλλου. Ένα προεπιλεγμένο RPC μπορεί να ανταγωνιστεί προσφέροντας πλουσιότερη ροή, αλλά οι χρήστες και οι ανταγωνιστικοί clients θα πρέπει να μπορούν να επιλέγουν απλούστερες, ιδιωτικές, χρονολογικές, χωρίς διαφημίσεις ή ειδικές για μια κοινότητα εναλλακτικές.

## RPC χωρίς θεματοφυλακή

Κάθε χρήστης θα πρέπει να μπορεί να συμμετέχει ως πλήρης peer-to-peer κόμβος μέσω RPC χωρίς να παραχωρεί στον πάροχο RPC την κυριότητα της ταυτότητας ή του προφίλ του.

Η φιλοξενούμενη διαδρομή είναι σημαντική, επειδή οι περισσότεροι χρήστες δεν θα ξεκινήσουν λειτουργώντας έναν διακομιστή. Η διαδρομή εξόδου είναι εξίσου σημαντική: ο χρήστης θα πρέπει να μπορεί να μεταφερθεί στον δικό του κόμβο προφίλ σε εξοπλισμό χαμηλών απαιτήσεων, συμπεριλαμβανομένου ενός Raspberry Pi, όποτε το επιθυμεί.

Αυτή είναι η διαφορά ανάμεσα στην ευκολία και τη θεματοφυλακή.

## Δημόσια συζήτηση, ενισχυμένη από το Bitsocial Chain

Το Bitsocial Chain μπορεί να ενσωματώσει σταθερή ονοματοδοσία, πληρωμές, φιλοδωρήματα, βραβεία και άλλες χρηματοοικονομικές δυνατότητες απευθείας στη δημόσια συζήτηση.

Ο client της Φάσης 3 παραμένει εστιασμένος στις αναρτήσεις, το κείμενο, την ανταλλαγή ιδεών και τη συζήτηση σε πραγματικό χρόνο, ενώ μοιράζεται κοινότητες και αποτελέσματα δικτύου με άλλους clients του Bitsocial.

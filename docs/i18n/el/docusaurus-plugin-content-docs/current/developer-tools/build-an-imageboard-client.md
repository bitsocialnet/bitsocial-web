# Δημιουργήστε ένα πρόγραμμα-πελάτη εικόνας

Η Φάση 1 δεν αφορά μία επίσημη εφαρμογή που καλύπτει ολόκληρη την κατηγορία. Το 5chan είναι το πρώτο σημείο απόδειξης, αλλά ο πραγματικός στόχος είναι ένα ευρύ οικοσύστημα εικόνας: πολλαπλοί πελάτες Bitsocial με διαφορετικές οπτικές γλώσσες, προεπιλογές εποπτείας, μοντέλα καταλόγου και κοινότητες στόχων.

## Τι χρειάζεται η Φάση 1

- Γνωστοί πελάτες σε στυλ 4chan για mainstream ενσωμάτωση
- Πελάτες εμπνευσμένοι από την Altchan με διαφορετικές κουλτούρες και πακέτα επιτραπέζιων επιτροπών
- Πελάτες που προορίζονται για κινητά ή χαμηλού εύρους ζώνης
- Πελάτες μεμονωμένων κοινοτήτων ή εξειδικευμένων πελατών με ισχυρές προεπιλογές
- Καλύτερες ροές εποπτείας, πολυμέσων, ενσωμάτωσης ή ανακάλυψης σε σχέση με την πρώτη εφαρμογή

## Ο πιο γρήγορος τρόπος για να βοηθήσετε

Εάν θέλετε τη συντομότερη διαδρομή προς την αποστολή, συνεισφέρετε πρώτα απευθείας στο 5chan:

- Εξερευνήστε τη ζωντανή εφαρμογή στο [5chan.app](https://5chan.app)
- Ελέγξτε την πηγή στο [github.com/bitsocialnet/5chan](https://github.com/bitsocialnet/5chan)
- Εγγραφείτε στη συνομιλία του οικοδόμου στο [t.me/fivechandev](https://t.me/fivechandev)

## Φτιάξτε τον δικό σας πελάτη

Εάν το 5chan δεν ταιριάζει με την κοινότητα ή τη διεπαφή που θέλετε, δημιουργήστε έναν ξεχωριστό πελάτη. Οι συμβατοί πελάτες Bitsocial μπορούν να μοιράζονται το ίδιο δίκτυο χωρίς να μοιράζονται τις ίδιες αποφάσεις προϊόντων.

1. Μάθετε τα εργαλεία που αντιμετωπίζουν το πρωτόκολλο:
   - [Bitsocial React hooks](../react-hooks/)
   - [Bitsocial CLI](../cli/)
2. Αποφασίστε τι είδους πίνακα εικόνας κατασκευάζετε πραγματικά.
Επιλέξτε πρώτα τη δομή του πίνακα, τις υποθέσεις ταυτότητας, το μοντέλο ελέγχου, τη ροή ανακάλυψης και την οπτική γλώσσα.
3. Επιλέξτε τη διαδρομή υλοποίησης που ταιριάζει στο έργο.
Fork 5chan αν θέλετε να κινηθείτε γρήγορα με μια γνωστή βάση εικόνας. Ξεκινήστε από την αρχή εάν η διεπαφή χρήστη ή το μοντέλο αλληλεπίδρασης πρέπει να είναι ριζικά διαφορετικό.
4. Αποστολή μιας στενής πρώτης έκδοσης.
Ένας πελάτης που εξυπηρετεί καλά μια πραγματική κοινότητα είναι πιο πολύτιμος από έναν αόριστο κλώνο που έχει σκοπό να ικανοποιήσει όλους.
5. Δημοσιεύστε το αποτέλεσμα και αφήστε τις κοινότητες να το δοκιμάσουν.
Το Bitsocial βελτιώνεται όταν εξωτερικοί κατασκευαστές στέλνουν πελάτες με γνώμη που ανταγωνίζονται στην ποιότητα των προϊόντων αντί να περιμένουν μια επίσημη εφαρμογή να κάνει τα πάντα.

## Αρχή σχεδιασμού

Το Bitsocial δεν κερδίζει έχοντας έναν ευλογημένο πελάτη. Κερδίζει όταν πολλοί πελάτες μπορούν να συνυπάρξουν, να διαχωρίσουν, να ειδικευτούν και να εξυπηρετήσουν ανάγκες που η πρώτη εφαρμογή δεν θα το κάνει ποτέ.

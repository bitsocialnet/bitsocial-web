# Pagkakakilanlan at Pagmamay-ari ng Komunidad

Ang mga profile at komunidad sa Bitsocial ay kinokontrol ng mga keypair, hindi ng mga maaaring bawiin na platform account.

Binabago nito ang relasyon sa pagitan ng pagmamay-ari at mga service provider. Maaari mong italaga ang uptime, pagho-host, o kaginhawahan sa isa pang serbisyo nang hindi binibigyan ang serbisyong iyon ng permanenteng pangangalaga sa pagkakakilanlan o komunidad mismo.

## Bakit mahalaga ang mga susi dito

Ang isang naka-host na platform account ay nirerentahan sa huli. Ito ay umiiral hangga't ang plataporma ay patuloy na parangalan ito.

Iba ang pagkakakilanlan na kinokontrol ng susi:

- ang pribadong susi ay ang pinagmumulan ng awtoridad
- maaaring palitan ang hosting
- maaaring magbago ang pagpili ng app
- ang pagmamay-ari ay hindi kailangang lumipat sa imprastraktura

## Ano ang ibig sabihin ng delegasyon

Ang delegasyon ay dapat mangahulugan ng tulong sa pagkakaroon, hindi pagsuko ng kontrol.

Halimbawa, maaaring panatilihin ng isang pampublikong RPC o serbisyo sa pagho-host ang isang node online, ngunit hindi dapat kailanganin ng serbisyong iyon ang kapangyarihan upang permanenteng sakupin ang profile o komunidad. Ang tulong ay kapaki-pakinabang. Ang pag-iingat ay dapat manatiling opsyonal.

## Ang mas malawak na punto

Ginagawa nitong mas malapit ang pagkakakilanlang Bitsocial sa isang asset na kinokontrol ng wallet kaysa sa isang username na nirentahan mula sa isang kumpanya. Maaaring palitan ang layer ng serbisyo. Ang relasyon ng may-ari ay dapat makaligtas sa pagpapalit na iyon.

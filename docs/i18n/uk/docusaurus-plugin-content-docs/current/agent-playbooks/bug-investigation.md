# Робочий процес дослідження помилок

Використовуйте це, коли повідомляється про помилку в певному файлі/рядку/блоці коду.

## Обов'язковий перший крок

Перед редагуванням перевірте історію git на відповідний код. Попередні учасники могли представити поведінку для крайнього випадку/обхідного шляху.

## робочий процес

1. Сканувати останні заголовки комітів (лише заголовки) для файлу/області:

```bash
# Заголовки останніх комітів для певного файлу
git log --oneline -10 -- src/components/post-desktop/post-desktop.tsx

# Останні назви комітів для певного діапазону рядків
git blame -L 120,135 src/components/post-desktop/post-desktop.tsx
```

2. Перевіряйте лише релевантні коміти з обмеженими відмінностями:

```bash
# Показати повідомлення коміту + diff для одного файлу
git show <commit-hash> -- path/to/file.tsx
```

3. Продовжуйте відтворення та виправте після розуміння контексту історії.

## Правило усунення несправностей

Якщо заблоковано, шукайте в Інтернеті останні виправлення/обхідні шляхи.

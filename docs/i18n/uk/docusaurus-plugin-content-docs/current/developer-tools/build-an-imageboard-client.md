# Створіть клієнт іміджборду

Перший етап не стосується однієї офіційної програми, яка охоплює всю категорію. 5chan є першим доказом, але справжньою метою є широка екосистема графічних дошок: кілька клієнтів Bitsocial з різними візуальними мовами, параметрами модерації за замовчуванням, моделями каталогів і цільовими спільнотами.

## Що потрібно для Фази 1

- Знайомі клієнти у стилі 4chan для основного навчання
- Клієнти, натхненні Альтчаном, з різними культурами та пакетами плат
- Мобільні клієнти або клієнти з низькою пропускною здатністю
- Клієнти з окремої спільноти або нішеві клієнти з твердими переконаннями за замовчуванням
- Кращі потоки модерування, медіа, адаптації та відкриття, ніж у першому додатку

## Найшвидший спосіб допомогти

Якщо вам потрібен найкоротший шлях до доставки, зробіть внесок безпосередньо в 5chan:

- Перегляньте живу програму на [5chan.app](https://5chan.app)
- Перегляньте джерело на [github.com/bitsocialnet/5chan](https://github.com/bitsocialnet/5chan)
- Приєднуйтесь до чату розробника на [t.me/fivechandev](https://t.me/fivechandev)

## Створіть власного клієнта

Якщо 5chan не відповідає потрібному вам співтовариству чи інтерфейсу, замість цього створіть окремий клієнт. Сумісні клієнти Bitsocial можуть спільно використовувати ту саму мережу, не користуючись однаковими рішеннями про продукт.

1. Вивчіть інструменти для роботи з протоколами:
   - [Хуки Bitsocial React](../react-hooks/)
   - [Bitsocial CLI](../cli/)
2. Вирішіть, який іміджборд ви насправді створюєте.
Спершу виберіть структуру дошки, припущення щодо ідентичності, модель модерації, потік відкриття та візуальну мову.
3. Виберіть шлях реалізації, який відповідає проекту.
Fork 5chan, якщо ви хочете швидко рухатися зі знайомою базою іміджборду. Почніть заново, якщо інтерфейс користувача або модель взаємодії мають кардинально змінитися.
4. Поставте вузьку першу версію.
Один клієнт, який добре обслуговує одну реальну спільноту, є ціннішим, ніж розпливчастий клон, призначений для задоволення всіх.
5. Опублікуйте результат і дозвольте спільнотам перевірити його.
Bitsocial покращується, коли сторонні розробники надсилають впевнених клієнтів, які змагаються за якість продукції, а не чекають, поки одна офіційна програма зробить усе.

## Принцип конструкції

Bitsocial не виграє, маючи одного благословенного клієнта. Це виграє, коли багато клієнтів можуть співіснувати, розділятися, спеціалізуватися та обслуговувати потреби, яких перша програма ніколи не зможе.

# 提交和发布格式

在提出或实施有意义的代码更改时使用此选项。

## 提交建议格式

- **标题：** 传统提交风格，简短，用反引号括起来。
- 使用 `perf` （不是 `fix`）进行性能优化。
- **描述：** 可选的 2-3 个非正式句子描述解决方案。简洁，技术性，没有要点。

例子：

> **提交标题：** `fix: correct date formatting in timezone conversion`
>
> 更新了 `date-utils.ts` 中的 `formatDate()` 以正确处理时区偏移。

## GitHub 问题建议格式

- **标题：** 尽可能短，用反引号括起来。
- **描述：** 2-3 个非正式句子描述问题（不是解决方案），就好像仍未解决一样。

例子：

> **GitHub 问题：**
>
> - **标题：** `Date formatting displays incorrect timezone`
> - **描述：**当用户查看来自不同地区的帖子时，评论时间戳显示不正确的时区。 `formatDate()` 函数不考虑用户的本地时区设置。

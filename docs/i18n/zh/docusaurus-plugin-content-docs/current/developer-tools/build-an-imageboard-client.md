# 构建图像板客户端

第一阶段并不是一个涵盖整个类别的官方应用程序。 5chan 是第一个证明点，但实际目标是一个广泛的图像板生态系统：具有不同视觉语言、审核默认值、目录模型和目标社区的多个 Bitsocial 客户端。

## 第一阶段需要什么

- 熟悉的 4chan 风格客户适合主流入职
- 受 Altchan 启发的具有不同文化和董事会套餐的客户
- 移动优先或低带宽客户端
- 具有强烈固执己见的默认行为的单一社区或利基客户
- 比第一个应用程序附带的更好的审核、媒体、入门或发现流程

## 最快的帮助方式

如果您想要最短的运输路径，请先直接向 5chan 做出贡献：

- 浏览 [5chan.app](https://5chan.app)
- 查看源代码：[github.com/bitsocialnet/5chan](https://github.com/bitsocialnet/5chan)
- 加入 [t.me/ Fivechandev](https://t.me/fivechandev)

## 建立您自己的客户端

如果 5chan 与您想要的社区或界面不匹配，请构建一个单独的客户端。兼容的 Bitsocial 客户端可以共享相同的网络，而无需共享相同的产品决策。

1. 了解面向协议的工具：
   - [Bitsocial React 钩子](../react-hooks/)
   - [Bitsocial CLI](../cli/)
2. 决定您实际构建的图像板类型。
首先选择董事会结构、身份假设、审核模型、发现流程和视觉语言。
3. 选择适合项目的实施路径。
如果您想使用熟悉的图像板底座快速移动，请使用 Fork 5chan。如果 UI 或交互模型需要完全不同，请重新开始。
4. 发布一个狭窄的第一个版本。
为一个真正的社区提供良好服务的一个客户比一个旨在满足所有人的模糊克隆更有价值。
5. 发布结果并让社区进行测试。
当外部开发者提供固执己见的客户并在产品质量上进行竞争而不是等待一个官方应用程序完成所有工作时，Bitsocial 就会得到改善。

## 设计原理

Bitsocial 并不会因为拥有一位幸运的客户而获胜。当许多客户端可以共存、分叉、专业化并满足第一个应用程序永远无法满足的需求时，它就会获胜。

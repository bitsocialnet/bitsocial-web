# 身份和社区所有权

Bitsocial 中的个人资料和社区由密钥对控制，而不是由可撤销的平台帐户控制。

这改变了所有权和服务提供商之间的关系。您可以将正常运行时间、托管或便利性委托给其他服务，而无需让该服务永久托管身份或社区本身。

## 为什么键在这里很重要

最终租用托管平台帐户。只要平台继续尊重它，它就存在。

密钥控制的身份是不同的：

- 私钥是权威的来源
- 主机可以更换
- 应用程序选择可能会改变
- 所有权不需要随基础设施转移

## 代表团应该是什么意思

授权应该意味着提供可用性方面的帮助，而不是放弃控制权。

例如，公共 RPC 或托管服务可以使节点保持在线，但该服务不需要永久获取配置文件或社区的能力。帮助很有用。监护权仍应是可选的。

## 更广泛的观点

这使得 Bitsocial 身份感觉更接近于钱包控制的资产，而不是从公司租用的用户名。服务层可以换出。所有者关系应该在交换后继续存在。

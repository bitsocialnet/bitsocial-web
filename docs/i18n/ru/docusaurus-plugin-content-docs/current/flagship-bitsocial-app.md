# Децентрализовать Twitter/X

Фаза 3 — это план по созданию целенаправленной децентрализованной альтернативы Twitter/X. В её центре находится публичное общение, где на первом месте текст: короткие посты, ответы, репосты, подписки, обсуждения в реальном времени и сообщества, а платформенный слой остаётся открытым.

Twitter/X по-прежнему определяется постами, текстом и обменом идеями. Клиент Фазы 3 должен конкурировать на уровне этого основного опыта и сделать его исключительно качественным.

Эта страница описывает направление продукта, а не окончательную спецификацию выпуска. Точный интерфейс, лента по умолчанию, рекламная модель, функции AI и рынок RPC могут меняться по мере развития протокола и первых приложений.

## Что это должно доказать

Клиент должен доказать, что социальная сеть на основе профилей может не становиться кастодиальной платформой:

- пользователи могут владеть личностями и профилями
- сообщества и узлы профилей могут оставаться одноранговыми
- сообщества могут переносить сетевые эффекты на клиентов Bitsocial
- Поставщики RPC могут сделать приложение удобным, не получая контроль над личностью или профилем пользователя
- алгоритмы подачи могут быть дополнительными услугами вместо закона платформы
- другие клиенты по-прежнему могут конкурировать за ту же сеть

Цель — создать как можно более сильный клиент для публичного общения и показать, насколько широкими могут быть возможности протокола.

## Знакомый по назначению, сменяемый по устройству

Стандартный опыт должен конкурировать с основой Twitter/X: быстрая главная лента, текстовые посты, подписки, ответы, распространение через репосты, сообщества, уведомления, поиск и ранжированная лента For You, которая работает сразу.

Bitsocial Forge может запустить первый RPC по умолчанию и службу подачи. По умолчанию это может включать ранжированный канал и рекламу, чтобы приложение выглядело завершенным с первого же дня, вместо того, чтобы просить обычных пользователей собрать весь стек самостоятельно.

Разница в том, что настройки по умолчанию не должны становиться тюрьмой. Пользователь должен иметь возможность менять RPC, ленты, инстансы, системы ранжирования, рекламу и сервисы обнаружения либо полностью отключать ранжирование. Клиент может предлагать чёткие начальные настройки, сохраняя при этом сменяемость каждой важной службы.

Благодаря этому клиент настраивается гибче обычной платформы. Один пользователь может оставить стандартную ранжированную ленту с рекламой. Другой — выбрать хронологическую ленту без ранжирования. Третий — использовать ориентированный на конфиденциальность RPC, сервис обнаружения от сообщества, платную ленту без рекламы или нишевый алгоритм для определённой субкультуры.

## Межклиентские сообщества

Сообщества должны быть гораздо важнее, чем изолированные группы внутри одного приложения.

В X/Twitter сообщества ограничены внутри X. Они могут быть полезны, но наследуют ограничения одной платформы, одной системы учетных записей, одного набора рекомендаций и одной поверхности продукта.

Сообщество Bitsocial можно создавать, размещать, находить и использовать через разные клиенты. Поэтому клиент Фазы 3 может показывать сообщества и посты из всей сети Bitsocial, а не только от пользователей, начавших работу внутри него. В одном сообществе может одновременно появляться активность из клиента имиджборда, дискуссионного клиента в стиле Reddit, клиента нишевого форума, мобильного приложения и клиента Фазы 3.

В этом заключается основное преимущество сетевого эффекта: один клиент может показаться знакомым обычным пользователям, сохраняя при этом ценность от многих клиентов, узлов сообщества, поставщиков RPC и независимых сервисов.

## Дополнительные алгоритмы подачи

Клиент Фазы 3 не должен навязывать всем единую глобальную систему ранжирования.

Алгоритмы подачи должны быть добровольными. Пользователь может выбрать алгоритм на торговой площадке, сменить поставщика, использовать алгоритм компании, использовать алгоритм анонимного оператора, использовать алгоритм, созданный сообществом, запустить личный алгоритм или вообще не использовать алгоритм.

Публичные поставщики RPC являются естественной площадкой для конкуренции этих услуг. Они могут индексировать, ранжировать и рекомендовать контент, но не должны контролировать личность или профиль пользователя.

Эти сервисы также могут конкурировать по форме самого приложения. Один RPC может предоставлять ранжированный канал с рекламой. Другой может предоставить неранжированную хронологическую ленту. Другой может специализироваться на конфиденциальности, переводе, модерации, открытии сообщества или нишевом социальном графе.

Если экономика сработает, службы каналов, поддерживаемые RPC, могут добавить функции искусственного интеллекта, аналогичные тем, которые основные платформы пытаются внедрить в свои каналы: автоматические переводы, резюме, ответы с помощью ботов, ответы на поиск, помощь в модерации или контекст в стиле комментариев сообщества.

Эти функции должны быть выбором сервиса, а не требованиями протокола. RPC по умолчанию может конкурировать, предлагая более богатый канал, но пользователи и конкурирующие клиенты по-прежнему должны иметь возможность выбирать более простые, частные, хронологические, без рекламы или специфичные для сообщества альтернативы.

## RPC без передачи контроля над личностью или профилем

Каждый пользователь должен иметь возможность участвовать в качестве полноценного однорангового узла через RPC, не передавая поставщику RPC право собственности на свою личность или профиль.

Размещенный путь имеет значение, поскольку большинство пользователей не начинают с запуска сервера. Путь выхода имеет не меньшее значение: пользователь должен иметь возможность перейти к своему собственному узлу профиля на оборудовании с низкими характеристиками, включая Raspberry Pi, когда захочет.

В этом разница между удобством и передачей контроля.

## Публичное общение, усиленное Bitsocial Chain

Bitsocial Chain может напрямую встроить в публичное общение постоянные имена, платежи, чаевые, награды и другие финансовые инструменты.

Клиент Фазы 3 сохраняет фокус на постах, тексте, обмене идеями и обсуждениях в реальном времени, одновременно разделяя сообщества и сетевые эффекты с другими клиентами Bitsocial.

# Децентрализовать все социальные сети

Это конечное состояние, описанное генеральным планом Bitsocial: не одно приложение, а экосистема приложений Bitsocial, которая может конкурировать со всеми основными категориями социальных сетей.

## Модель

В предлагаемом зрелом состоянии сети:

- несколько общедоступных поставщиков RPC конкурируют за хостинг узлов профиля
- пользователи по-прежнему могут самостоятельно размещать
- алгоритмы подачи становятся дополнительными услугами вместо обязательного закона о платформах
- монетизация не зависит от желания банков поддерживать платформу
- приложения конкурируют по качеству продукта, открытию и дизайну интерфейса.

## Как будет выглядеть успех

Приложения Bitsocial должны быть в состоянии бросить вызов платформам разных форматов, а не только в одной нише:

- текст и обсуждение
- имиджборды
- форумы
- сообщества авторов
- короткие видеоролики и более широкие медиапродукты

Цель не в том, чтобы навязать всем один универсальный интерфейс. Целью этой программы является предоставление многим приложениям доступа к одним и тем же базовым сообществам, идентификаторам и сети.

## Почему здесь важен уровень RPC

Постоянно активные сообщества и профили становятся гораздо более практичными, когда существуют публичные рынки RPC. Без этого уровня самостоятельный хостинг остается слишком сложным для многих пользователей, особенно на мобильных устройствах.

## Почему здесь важен сетевой уровень

Если монетизация контента может быть прекращена обычным банковским давлением или давлением платформы, то экосистема останется хрупкой. Битсоциальная сеть предлагается в качестве экономического слоя, который делает долгосрочную конкуренцию более надежной.

## Основная ставка

Большая ставка заключается в том, что социальные сети можно разбить на заменяемые слои:

- протокол
- сообщества
- личность
- хостинг
- открытие
- монетизация
- приложения

Как только эти слои станут заменяемыми, ни одной компании не придется владеть всем социальным графом, чтобы продукт работал.

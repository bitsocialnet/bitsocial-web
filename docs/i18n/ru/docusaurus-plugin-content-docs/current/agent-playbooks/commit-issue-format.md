# Формат фиксации и выпуска

Используйте это при предложении или реализации значимых изменений кода.

## Формат предложения по фиксации

- **Название:** Традиционный стиль фиксации, короткий, заключенный в обратные кавычки.
- Используйте `perf` (а не `fix`) для оптимизации производительности.
- **Описание:** Необязательные 2–3 неформальных предложения, описывающих решение. Кратко, технично, без пунктов.

Пример:

> **Заголовок фиксации:** `fix: correct date formatting in timezone conversion`
>
> Обновлен `formatDate()` в `date-utils.ts` для правильной обработки смещений часовых поясов.

## Формат предложения по проблеме GitHub

- **Название:** Как можно короче, в обратных кавычках.
- **Описание:** 2–3 неформальных предложения, описывающих проблему (а не решение), как будто она еще не решена.

Пример:

> **Проблема на GitHub:**
>
> - **Название:** `Date formatting displays incorrect timezone`
> - **Описание:** В временных метках комментариев отображаются неправильные часовые пояса, когда пользователи просматривают сообщения из разных регионов. Функция `formatDate()` не учитывает настройки местного часового пояса пользователя.

# 5chan

5chan एक सर्व्हरलेस, ॲडमिनलेस आणि पूर्णपणे विकेंद्रित इमेजबोर्ड आहे जो बिटसोशियल प्रोटोकॉलवर चालतो. विकेंद्रित मालकी सादर करताना हे परिचित इमेजबोर्ड निर्देशिकेच्या संरचनेचे अनुसरण करते — कोणीही बोर्ड तयार करू शकतो आणि मतदान यंत्रणेद्वारे एकाच निर्देशिका स्लॉटसाठी एकाधिक बोर्ड स्पर्धा करू शकतात.

## डाउनलोड

| प्लॅटफॉर्म | लिंक                               |
| ---------- | ---------------------------------- |
| वेब        | [5chan.app](https://5chan.app)     |
| डेस्कटॉप   | Mac, Windows आणि Linux साठी उपलब्ध |
| मोबाईल     | Android साठी उपलब्ध                |

## बोर्ड कसे कार्य करतात

5chan क्लासिक डिरेक्टरी लेआउट (उदा. `/b/`, `/g/`) वापरून बोर्डमध्ये सामग्री आयोजित करते. पारंपारिक इमेजबोर्डच्या विपरीत जेथे केंद्रीय प्रशासक प्रत्येक बोर्ड नियंत्रित करतो, 5chan कोणत्याही वापरकर्त्याला त्यांचे स्वतःचे बोर्ड तयार करण्याची आणि पूर्ण मालकीची परवानगी देतो. जेव्हा अनेक बोर्ड एकाच डिरेक्टरी स्लॉटला लक्ष्य करतात, तेव्हा ते मतदानाद्वारे त्या स्थानासाठी स्पर्धा करतात.

### बोर्ड तयार करणे

नवीन बोर्ड तयार करण्यासाठी, तुम्हाला पीअर-टू-पीअर नोड म्हणून `bitsocial-cli` चालवावे लागेल. हे सुनिश्चित करते की तुमचा बोर्ड कोणत्याही केंद्रीय सर्व्हरवर अवलंबून न राहता विकेंद्रित पद्धतीने होस्ट केला जातो.

### निर्देशिका असाइनमेंट

डिरेक्टरी स्लॉट असाइनमेंट (कोणता बोर्ड कोणत्या मार्गावर दिसतो) सध्या GitHub द्वारे `5chan-directories.json` फाईलच्या पुल विनंत्या व्यवस्थापित केल्या जातात. ही एक तात्पुरती प्रक्रिया आहे — भविष्यातील रिलीझ निर्देशिका असाइनमेंट स्वयंचलितपणे हाताळण्यासाठी ॲप-मधील बोर्ड निर्मिती आणि पबसब-आधारित मतदानास समर्थन देतील.

## अंतर्गत

हूड अंतर्गत, 5chan त्याच्या नेटवर्क परस्परसंवादासाठी सामायिक बिटसोशियल प्रोटोकॉल क्लायंट लेयर वापरते. 5chan.app वरील वेब ॲप प्रगत सेटिंग्जमधून ब्राउझर P2P सक्षम केलेले असताना ब्राउझरमध्ये हेलिया नोड देखील चालवू शकते, त्यामुळे वाचक केंद्रीकृत IPFS गेटवेशिवाय समवयस्कांकडून लोड करू शकतात. पीअर-टू-पीअर प्रोटोकॉल नोट्समधील ब्राउझर P2P विभाग पहा.

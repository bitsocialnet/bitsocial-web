# EVM Contract Call Challenge

:::warning वारसा नामकरण
हे पॅकेज मूलतः `@plebbit` स्कोप अंतर्गत प्रकाशित केले गेले. त्याचे `@bitsocial/evm-contract-challenge` असे नामकरण करण्यात आले आहे. जुन्या नावाचे संदर्भ अजूनही जुन्या कागदपत्रांमध्ये किंवा कोडबेसमध्ये दिसू शकतात.
:::

ईव्हीएम कॉन्ट्रॅक्ट कॉल चॅलेंज ही स्पॅमविरोधी यंत्रणा आहे जी प्रकाशनाला परवानगी देण्यापूर्वी ऑन-चेन परिस्थितीची पडताळणी करते. मूळतः `plebbit-js` वरून स्वतंत्र पॅकेज म्हणून काढलेले, ते समुदाय मालकांना लेखकांना स्मार्ट-करार-परिभाषित निकषांची पूर्तता करणे आवश्यक आहे -- उदाहरणार्थ, किमान टोकन शिल्लक धारण -- पोस्ट करण्यासाठी.

**स्रोत कोड:** [github.com/bitsocialnet/evm-contract-call](https://github.com/bitsocialnet/evm-contract-call)

## आवश्यकता

- **Node.js** >= २२
- **फक्त ईएसएम** -- हे पॅकेज CommonJS बिल्ड पाठवत नाही.
- **रनटाइम पीअर अवलंबित्व:** `@plebbit/plebbit-js` (`@pkc/pkc-js` वर स्थलांतरित करत आहे)

## स्थापना

```bash
npm install @bitsocial/evm-contract-challenge
```

## कॉन्फिगरेशन पर्याय

| पर्याय        | प्रकार   | वर्णन                                                                                      |
| ------------- | -------- | ------------------------------------------------------------------------------------------ |
| `chainTicker` | `string` | क्वेरी करण्यासाठी साखळी (उदा., `eth`, `matic`, `avax`).                                    |
| `address`     | `string` | कॉल करण्यासाठी स्मार्ट करार पत्ता.                                                         |
| `abi`         | `string` | कॉल केलेल्या कार्यासाठी ABI खंड.                                                           |
| `condition`   | `string` | कॉन्ट्रॅक्ट रिटर्न व्हॅल्यू (उदा. `> 1000`) विरुद्ध मूल्यमापन केलेली तुलनात्मक अभिव्यक्ती. |
| `error`       | `string` | अट पूर्ण न करणाऱ्या लेखकांना दाखवलेला त्रुटी संदेश.                                        |

## उदाहरण

एखादा समुदाय मालक जो विशिष्ट ERC-20 टोकन पैकी 1,000 पेक्षा जास्त लेखकांना पोस्ट करणे प्रतिबंधित करू इच्छितो तो यासह आव्हान कॉन्फिगर करेल:

- `chainTicker`: `"eth"`
- `address`: टोकन कराराचा पत्ता
- `abi`: `balanceOf(address)` साठी ABI
- `condition`: `"> 1000"`
- `error`: `"You must hold more than 1,000 tokens to post in this community."`

जेव्हा लेखक प्रकाशित करण्याचा प्रयत्न करतो, तेव्हा आव्हान लेखकाच्या पत्त्यासह `balanceOf` वर कॉल करते आणि परत केलेले मूल्य अट पूर्ण करते की नाही ते तपासते. तसे झाल्यास, प्रकाशन पुढे जाईल; अन्यथा, कॉन्फिगर केलेला त्रुटी संदेश परत येईल.

## ते कधी वापरायचे

ईव्हीएम कॉन्ट्रॅक्ट कॉल चॅलेंज यासाठी आदर्श आहे:

- **टोकन-गेट केलेले समुदाय** जे टोकनधारकांना पोस्ट करणे प्रतिबंधित करतात.
- **NFT-गेट केलेला प्रवेश** जेथे विशिष्ट NFT ची मालकी आवश्यक आहे.
- **DAO गव्हर्नन्स स्पेस** जिथे सहभाग गव्हर्नन्स टोकन धारकांपुरता मर्यादित आहे.

ऑन-चेन ओळखीवर अवलंबून नसलेल्या समुदायांसाठी, त्याऐवजी [Spam Blocker](./spam-blocker.md) किंवा [Voucher Challenge](./voucher-challenge.md) विचारात घ्या.

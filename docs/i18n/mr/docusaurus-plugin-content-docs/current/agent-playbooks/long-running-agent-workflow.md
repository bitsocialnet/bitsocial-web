# दीर्घकाळ चालणारा एजंट वर्कफ्लो

हे प्लेबुक वापरा जेव्हा एखादे कार्य एकाधिक सत्रे, हँडऑफ किंवा स्पॉन केलेल्या एजंट्सपर्यंत पोहोचण्याची शक्यता असते.

## गोल

- प्रत्येक ताज्या सत्राला संदर्भ पुन्हा मिळवण्याचा जलद मार्ग द्या
- मोठ्या बदलाचे एक-शॉटिंग करण्याऐवजी काम वाढवत ठेवा
- आणखी कोड जोडण्यापूर्वी तुटलेली लोकल बेसलाइन पकडा
- पुढील सत्रावर विश्वास ठेवू शकतील अशा टिकाऊ कलाकृती सोडा

## राज्य कुठे ठेवावे

- जेव्हा मानव, पुनरावलोकन बॉट्स किंवा एकाधिक टूलचेनला समान कार्य स्थिती आवश्यक असेल तेव्हा `docs/agent-runs/<slug>/` वापरा.
- टूल-लोकल डिरेक्ट्रीचा वापर करा जसे की `.codex/runs/<slug>/` फक्त जेव्हा टास्क स्टेट एखाद्या वर्कस्टेशनवर किंवा एका टूलचेनसाठी हेतुपुरस्सर स्थानिक असते.
- दुसऱ्या योगदानकर्त्याला किंवा एजंटला नंतर त्याची आवश्यकता भासल्यास खाजगी स्क्रॅच फाइलमध्ये मल्टी-सेशन शेअर केलेली स्थिती लपवू नका.

## आवश्यक फाइल्स

दीर्घकाळ चालणाऱ्या कार्याच्या सुरुवातीला या फायली तयार करा:

- `feature-list.json`
- `progress.md`

`docs/agent-playbooks/templates/feature-list.template.json` आणि `docs/agent-playbooks/templates/progress.template.md` मध्ये टेम्पलेट्स वापरा.

वैशिष्ट्य सूचीसाठी JSON ला प्राधान्य द्या जेणेकरून एजंट संपूर्ण दस्तऐवज पुन्हा न लिहिता काही फील्ड अद्यतनित करू शकतील.

## सत्र प्रारंभ चेकलिस्ट

1. `pwd` चालवा.
2. `progress.md` वाचा.
3. `feature-list.json` वाचा.
4. `git log --oneline -20` चालवा.
5. `./scripts/agent-init.sh --smoke` चालवा.
6. `pending`, `in_progress`, किंवा `blocked` असलेला एक सर्वोच्च-प्राथमिकता आयटम निवडा.

स्मोक स्टेप अयशस्वी झाल्यास, नवीन वैशिष्ट्य स्लाइस लागू करण्यापूर्वी तुटलेली बेसलाइन दुरुस्त करा.

## सत्राचे नियम

- एका वेळी एका वैशिष्ट्यावर किंवा टास्क स्लाइसवर कार्य करा.
- वैशिष्ट्य सूची मशीन-वाचनीय आणि स्थिर ठेवा. असंबंधित आयटम पुन्हा लिहिण्याऐवजी स्थिती, नोट्स, फाइल्स आणि सत्यापन फील्ड अद्यतनित करा.
- आदेश चालवल्यानंतर किंवा त्या आयटममध्ये सूचीबद्ध केलेल्या वापरकर्ता प्रवाहानंतर सत्यापित केलेल्या आयटमवर फक्त चिन्हांकित करा.
- बाउंडेड स्लाइससाठी स्पॉन केलेले एजंट वापरा, संपूर्ण टास्क-स्टेट मालकीसाठी नाही.
- जेव्हा लहान एजंटकडे एक आयटम असतो, तेव्हा त्याला अचूक आयटम आयडी, स्वीकृती निकष आणि तो स्पर्श करू शकतील अशा फाइल्स द्या.

## सत्र समाप्ती चेकलिस्ट

1. `progress.md` वर एक छोटी प्रगती एंट्री जोडा.
2. स्पर्श केलेला आयटम `feature-list.json` मध्ये अपडेट करा.
3. पडताळणीसाठी चालवलेल्या अचूक कमांड्स रेकॉर्ड करा.
4. कॅप्चर ब्लॉकर्स, फॉलो-अप आणि पुन्हा सुरू करण्यासाठी पुढील सर्वोत्तम आयटम.

# ज्ञात आश्चर्ये

ही फाइल रेपॉजिटरी-विशिष्ट गोंधळाच्या बिंदूंचा मागोवा घेते ज्यामुळे एजंटच्या चुका झाल्या.

## प्रवेश निकष

सर्व सत्य असल्यासच प्रविष्टी जोडा:

- हे या भांडारासाठी विशिष्ट आहे (सामान्य सल्ला नाही).
- भविष्यातील एजंटसाठी पुनरावृत्ती होण्याची शक्यता आहे.
- त्यात एक ठोस शमन आहे ज्याचे अनुसरण केले जाऊ शकते.

अनिश्चित असल्यास, एंट्री जोडण्यापूर्वी विकासकाला विचारा.

## टेम्पलेट

```md
### [Short title]

- **Date:** YYYY-MM-DD
- **Observed by:** agent name or contributor
- **Context:** where/when it happened
- **What was surprising:** concrete unexpected behavior
- **Impact:** what went wrong or could go wrong
- **Mitigation:** exact step future agents should take
- **Status:** confirmed | superseded
```

### पोर्टलेस कॅनॉनिकल स्थानिक ॲप URL बदलते

- **तारीख:** 2026-03-18
- **निरीक्षण केलेले:** कोडेक्स
- **संदर्भ:** ब्राउझर सत्यापन आणि धूर प्रवाह
- **काय आश्चर्यकारक होते:** डीफॉल्ट स्थानिक URL नेहमीच्या Vite पोर्ट नाही. रेपोला पोर्टलेसद्वारे `http://bitsocial.localhost:1355` अपेक्षित आहे, त्यामुळे `localhost:3000` किंवा `localhost:5173` तपासणे चुकीचे ॲप किंवा काहीही नाही.
- **प्रभाव:** ब्राउझर तपासणी अयशस्वी होऊ शकते किंवा डेव्हल सर्व्हर निरोगी असताना देखील चुकीचे लक्ष्य प्रमाणित करू शकते.
- **शमन:** प्रथम `http://bitsocial.localhost:1355` वापरा. जेव्हा तुम्हाला थेट Vite पोर्टची आवश्यकता असेल तेव्हाच `PORTLESS=0 corepack yarn start` सह बायपास करा.
- **स्थिती:** पुष्टी केली

### कमिटिझन हुक नॉन-इंटरॅक्टिव्ह कमिट ब्लॉक करतात

- **तारीख:** 2026-03-18
- **निरीक्षण केलेले:** कोडेक्स
- **संदर्भ:** एजंट-चालित कमिट वर्कफ्लो
- **काय आश्चर्यकारक होते:** `git commit` हस्की द्वारे कमिटीझन ट्रिगर करते आणि परस्परसंवादी TTY इनपुटची प्रतीक्षा करते, जे नॉन-इन्टरॅक्टिव हँग करते. shells.
- **प्रभाव:** एजंट सामान्य कमिट दरम्यान अनिश्चित काळासाठी थांबू शकतात.
- **शमन:** एजंटने तयार केलेल्या कमिटसाठी `git commit --no-verify -m "message"` वापरा. लोक अजूनही `corepack yarn commit` किंवा `corepack yarn exec cz` वापरू शकतात.
- **स्थिती:** पुष्टी केली

- **तारीख:** 2026-03-19

- **प्रभाव:** विकसक चुकून रेपोच्या पॅकेज-व्यवस्थापक पिनिंगला बायपास करू शकतात आणि भिन्न इंस्टॉल वर्तन किंवा लॉकफाइल आउटपुट मिळवू शकतात.
- **शमन:** शेल कमांडसाठी `corepack yarn ...` वापरा, किंवा `corepack yarn ...` चालवा यार्न 4 आवृत्ती.

### फिक्स्ड पोर्टलेस ॲपची नावे बिटसोशल वेब वर्कट्रीजवर आदळली आहेत

- **तारीख:** 2026-03-30
- **निरीक्षण केलेले:** कोडेक्स
- **संदर्भ:** `yarn start` एक बिटसोशियल वेब वर्कट्री द्वारे आधीच सुरू करत असताना दुसऱ्या कामात पोर्टलेस
- **काय आश्चर्यकारक होते:** प्रत्येक वर्कट्रीमध्ये अक्षरशः पोर्टलेस ॲप नाव `bitsocial` वापरल्याने मार्ग स्वतःच आदळतो, जरी बॅकिंग पोर्ट भिन्न असतात, त्यामुळे दुसरी प्रक्रिया अयशस्वी होते कारण `bitsocial.localhost` आधीच नोंदणीकृत आहे. समांतर शाखा देखील एकमेकींना ब्लॉक करू शकतात. Portless त्यांना सुरक्षितपणे एकत्र राहू देण्यासाठी आहे.
- **शमन:** पोर्टलेस स्टार्टअपला `scripts/start-dev.mjs` च्या मागे ठेवा, जे आता कॅनॉनिकल केसच्या बाहेर शाखा-स्कोप असलेला `*.bitsocial.localhost:1355` मार्ग वापरते आणि bare `bitsocial.localhost` नाव आधीच व्यापलेले असेल तेव्हा शाखा-स्कोप असलेल्या मार्गावर परत येते.
- **स्थिती:** पुष्टी केली

### हार्ड-कोड पोर्ट 3001 साठी वापरलेले डॉक्स पूर्वावलोकन

- **तारीख:** 2026-03-30
- **निरीक्षण केलेले:** कोडेक्स
- **संदर्भ:** इतर स्थानिक रेपो आणि एजंटसह `yarn start` चालवणे
- **काय आश्चर्यकारक होते:** रूट डेव्ह कमांडने `docusaurus start --port 3001` सह डॉक्स वर्कस्पेस चालवले, त्यामुळे जेव्हा मुख्य ॲप आधीच पोर्टलेस वापरला असला तरीही, `3001` च्या मालकीची दुसरी प्रक्रिया असताना संपूर्ण डेव्ह सेशन अयशस्वी झाले.
- **प्रभाव:** `3001` दुसऱ्या प्रक्रियेच्या ताब्यात होताच docs workspace देखील त्याच्या मालकीची झाली, त्यामुळे docs-port टक्करांमुळे असंबंधित स्थानिक काम लगेच अडथळले.
- **शमन:** डॉक्स स्टार्टअपला `yarn start:docs` च्या मागे ठेवा, जे आता इंजेक्टेड फ्री पोर्टचा सन्मान करण्यासाठी पोर्टलेस प्लस `scripts/start-docs.mjs` वापरते किंवा थेट चालवताना पुढील उपलब्ध पोर्टवर परत येते.
- **स्थिती:** पुष्टी केली

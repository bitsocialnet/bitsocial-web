# इमेजबोर्ड क्लायंट तयार करा

पहिला टप्पा संपूर्ण श्रेणीचा समावेश असलेल्या एका अधिकृत ॲपबद्दल नाही. 5chan हा पहिला पुरावा बिंदू आहे, परंतु वास्तविक लक्ष्य एक विस्तृत इमेजबोर्ड इकोसिस्टम आहे: भिन्न व्हिज्युअल भाषा, मॉडरेशन डीफॉल्ट, निर्देशिका मॉडेल आणि लक्ष्य समुदायांसह एकाधिक बिटसोशियल क्लायंट.

## पहिल्या टप्प्याला काय आवश्यक आहे

- मेनस्ट्रीम ऑनबोर्डिंगसाठी परिचित 4chan-शैलीतील क्लायंट
- विविध संस्कृती आणि बोर्ड बंडल असलेले Altchan-प्रेरित क्लायंट
- मोबाइल-प्रथम किंवा कमी-बँडविड्थ क्लायंट
- एकल-समुदाय किंवा मजबूत अभिमत डीफॉल्टसह विशिष्ट क्लायंट
- प्रथम ॲप शिप करण्यापेक्षा चांगले नियंत्रण, मीडिया, ऑनबोर्डिंग किंवा शोध प्रवाह

## मदत करण्याचा सर्वात जलद मार्ग

तुम्हाला शिपिंगसाठी सर्वात लहान मार्ग हवा असल्यास, प्रथम थेट 5chan मध्ये योगदान द्या:

- थेट ॲप एक्सप्लोर करा [5chan.app](https://5chan.app)
- [github.com/bitsocialnet/5chan](https://github.com/bitsocialnet/5chan)) येथे स्त्रोताचे पुनरावलोकन करा
- [t.me/fivechandev](https://t.me/fivechandev)) येथे बिल्डर चॅटमध्ये सामील व्हा

## तुमचा स्वतःचा क्लायंट तयार करा

5chan तुम्हाला हव्या असलेल्या समुदायाशी किंवा इंटरफेसशी जुळत नसल्यास, त्याऐवजी वेगळा क्लायंट तयार करा. सुसंगत बिटसोशियल क्लायंट समान उत्पादन निर्णय सामायिक न करता समान नेटवर्क सामायिक करू शकतात.

1. प्रोटोकॉल-फेसिंग टूल्स जाणून घ्या:
   - [Bitsocial React हुक](../react-hooks/)
   - [बिटसोल CLI](../cli/)
2. तुम्ही प्रत्यक्षात कोणत्या प्रकारचे इमेजबोर्ड बनवत आहात ते ठरवा.
प्रथम बोर्ड रचना, ओळख गृहीतक, नियंत्रण मॉडेल, शोध प्रवाह आणि दृश्य भाषा निवडा.
3. प्रकल्पाला बसणारा अंमलबजावणीचा मार्ग निवडा.
जर तुम्हाला परिचित इमेजबोर्ड बेससह जलद हलवायचे असेल तर Fork 5chan. UI किंवा परस्परसंवाद मॉडेल पूर्णपणे भिन्न असणे आवश्यक असल्यास नवीन प्रारंभ करा.
4. एक अरुंद पहिली आवृत्ती पाठवा.
एक क्लायंट जो एका खऱ्या समुदायाला चांगल्या प्रकारे सेवा देतो तो अस्पष्ट क्लोनपेक्षा अधिक मौल्यवान असतो ज्याचा अर्थ सर्वांना संतुष्ट करतो.
5. निकाल प्रकाशित करा आणि समुदायांना त्याची चाचणी करू द्या.
सर्व काही करण्यासाठी एका अधिकृत ॲपची वाट पाहण्याऐवजी उत्पादनाच्या गुणवत्तेवर स्पर्धा करणाऱ्या ग्राहकांना बाहेरचे बिल्डर पाठवतात तेव्हा बिटसोशियल सुधारते.

## डिझाइन तत्त्व

एक आशीर्वादित ग्राहक घेऊन बिटसोशियल जिंकत नाही. जेव्हा अनेक क्लायंट एकत्र राहू शकतात, फोर्क करू शकतात, स्पेशलायझेशन करू शकतात आणि सेवा देऊ शकतात तेव्हा प्रथम ॲप कधीही करणार नाही अशा गरजा देऊ शकतात.

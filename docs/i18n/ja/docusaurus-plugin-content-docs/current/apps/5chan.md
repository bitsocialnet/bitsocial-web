# 5chan

5chan は、Bitsocial プロトコルで実行される、サーバーレス、管理者不要の完全分散型イメージボードです。これは、分散型所有権を導入しながら、おなじみのイメージボード ディレクトリ構造に従います。誰でもボードを作成でき、投票メカニズムを通じて複数のボードが同じディレクトリ スロットを競うことができます。

## ボードの仕組み

5chan は、古典的なディレクトリ レイアウト (例: `/b/`、`/g/`) を使用してコンテンツをボードに整理します。従来とは異なります。中央管理者がすべてのボードを管理するイメージボードでは、5chan では、すべてのユーザーが独自のボードを作成して完全に所有することができます。複数のボードが同じディレクトリ スロットをターゲットにしている場合、投票を通じてそのポジションを競います。

### ボードの作成

新しいボードを作成するには、`bitsocial-cli` をピアツーピア ノードとして実行する必要があります。これにより、中央サーバーに依存せずにボードが分散型でホストされるようになります。

### ディレクトリ割り当て

ディレクトリ スロット割り当て (どのボードがどのパスに表示されるか) は現在、`5chan-directories.json` ファイルへの GitHub プル リクエストを通じて管理されています。これは一時的なプロセスです。将来のリリースでは、アプリ内ボードの作成と、ディレクトリ割り当てを自動的に処理する pubsub ベースの投票がサポートされる予定です。

## 内部

内部では、5chan はプロトコル インタラクションに plebbit-js API レイヤーを使用しています。上記の警告に記載されているように、これらの内部参照には、Bitsocial のブランド変更以前の古い名前がまだ使用されています。

# コミットと発行の形式

意味のあるコード変更を提案または実装する場合にこれを使用します。

## コミット提案の形式

- **タイトル:** 従来のコミット スタイル、短く、バックティックで囲みます。
- パフォーマンスのために `perf` (`fix` ではありません) を使用しますoptimizations.
- **説明:** ソリューションを説明するオプションの 2 ～ 3 つの非公式な文。簡潔で技術的で、箇条書きはありません。

例:

> **コミット タイトル:** `fix: correct date formatting in timezone conversion`
>
> タイムゾーン オフセットを適切に処理するために、`date-utils.ts` の `formatDate()` を更新しました。

## GitHub の問題の提案形式

- **タイトル:** できるだけ短く、バッククォートで囲みます。
- **説明:** 問題 (解決策ではない) を、未解決であるかのように説明する 2 ～ 3 の非公式な文。

例:

> **GitHub の問題:**
>
> - **タイトル:** `Date formatting displays incorrect timezone`
> - **説明:** ユーザーが異なる地域からの投稿を表示すると、コメントのタイムスタンプに誤ったタイムゾーンが表示されます。 `formatDate()` 関数は、ユーザーのローカル タイムゾーン設定を考慮しません。

# イメージボードクライアントを構築する

フェーズ 1 は、カテゴリ全体をカバーする 1 つの公式アプリではありません。 5chan は最初の実証ポイントですが、実際の目標は、さまざまなビジュアル言語、モデレーションのデフォルト、ディレクトリ モデル、ターゲット コミュニティを備えた複数の Bitsocial クライアントといった広範なイメージボード エコシステムです。

## フェーズ 1 に必要なもの

- 主流のオンボーディングのための使い慣れた 4chan スタイルのクライアント
- さまざまな文化とボード バンドルを持つ Altchan に影響を受けたクライアント
- モバイルファーストまたは低帯域幅のクライアント
- 独自の強いデフォルト設定を持つ単一コミュニティまたはニッチなクライアント
- 最初のアプリに同梱されているものよりも優れたモデレーション、メディア、オンボーディング、またはディスカバリーフロー

## 一番早い助け方

最短で発送したい場合は、まず 5chan に直接投稿してください。

- [5chan.アプリ](https://5chan.app)) でライブアプリを探索してください
- [github.com/bitsocialnet/5chan](https://github.com/bitsocialnet/5chan)) でソースを確認してください。
- [t.me/fivechandev](https://t.me/fivechandev)) でビルダー チャットに参加してください。

## 独自のクライアントを構築する

5chan が必要なコミュニティやインターフェイスと一致しない場合は、代わりに別のクライアントを構築してください。互換性のある Bitsocial クライアントは、同じ製品決定を共有することなく、同じネットワークを共有できます。

1. プロトコル対応ツールについて学習します。
   - [Bitsocial の React フック](../react-hooks/)
   - [ビットソーシャル CLI](../cli/)
2. 実際に構築するイメージボードの種類を決定します。
まず、ボードの構造、アイデンティティの前提、モデレーション モデル、発見フロー、および視覚的な言語を選択します。
3. プロジェクトに適した実装パスを選択してください。
使い慣れた画像掲示板ベースで高速に移動したい場合は、5chan をフォークしてください。 UI またはインタラクション モデルを根本的に変える必要がある場合は、最初からやり直してください。
4. ナローの最初のバージョンを出荷します。
1 つの実際のコミュニティに適切にサービスを提供する 1 つのクライアントは、全員を満足させることを目的とした漠然としたクローンよりも価値があります。
5. 結果を公開し、コミュニティでテストしてもらいます。
外部のビルダーが、1 つの公式アプリがすべてを行うのを待つのではなく、製品の品質で競争する独自のクライアントを出荷すると、Bitsocial は改善されます。

## 設計原理

Bitsocial は、恵まれたクライアントを 1 人抱えているだけでは勝ちません。多くのクライアントが共存し、フォークし、特化し、最初のアプリでは決してできなかったニーズに応えることができれば、成功します。

# Voucher Challenge

Voucher Challenge は、固有のバウチャー コードの背後でコンテンツの公開をゲートするスパム対策メカニズムです。自動検出に依存するのではなく、コミュニティ オーナーに信頼を移し、コミュニティ オーナーが信頼できる人にコードを手動で配布します。

**ソース コード:** [github.com/bitsocialnet/voucher-challenge](https://github.com/bitsocialnet/voucher-challenge)

## 仕組み

1. コミュニティ オーナーは 1 つ以上の一意のクーポン コードを生成します。
2. オーナーは、そのコードをそのチャネルを通じて信頼できる作成者に配布します。選択肢（ダイレクト メッセージ、電子メール、対面など）。
3. 著者が出版しようとすると、チャレンジ システムがバウチャー コードの入力を求めます。
4. コードは検証され、本物でまだ使用されていない場合、出版は受け入れられます。

各バウチャー コードは、引き換えられると特定の著者に関連付けられ、他の人による再利用を防ぎます。

## いつ使用するか

バウチャー チャレンジが最適です。対象:

- **メンバーシップが意図的に制限されている招待専用コミュニティ**。
- **オーナーがすべての参加者を個人的に精査するキュレーションされたスペース**。
- **自動スパム スコアリングが不要または望ましくない環境**。

手動によるコード配布が必要なため、大規模なオープン コミュニティには拡張できません。これらのシナリオの場合は、次のことを検討してください。代わりに、[Spam Blocker](./spam-blocker.md) または [EVM Contract Call Challenge](./evm-contract-call.md) を使用してください。

## 統合

バウチャー チャレンジは、Bitsocial エコシステム内の他のスパム対策パッケージで使用されているのと同じチャレンジ インターフェイスにプラグインされ、コミュニティの所有者はコミュニティ設定を通じてこれを有効にし、作成者が投稿しようとするとチャレンジが自動的に表示されます。

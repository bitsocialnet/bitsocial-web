# Spam Blocker

:::warning 従来の命名
このパッケージは当初、`@plebbit` スコープで公開されました。名前が `@bitsocial/spam-blocker-server` および `@bitsocial/spam-blocker-challenge` に変更されました。古い名前への参照は、古いドキュメントやコードベースにまだ記載されている場合があります。
:::

スパム ブロッカーは、受信したパブリケーションを評価してリスク スコアを割り当てる集中スパム検出サービスです。これは 2 つのパッケージで構成されます:

- **`@bitsocial/spam-blocker-server`** -- 評価 API とチャレンジ API をホストする HTTP サーバー。
- **`@bitsocial/spam-blocker-challenge`** -- 評価用のパブリケーションを送信するためにコミュニティが統合する軽量クライアント パッケージ。

**ソース コード:** [github.com/bitsocialnet/spam-blocker](https://github.com/bitsocialnet/spam-blocker)

## リスク スコアリングの仕組み

`/evaluate` エンドポイントに送信されたすべてのパブリケーションは、数値リスク スコアを受け取ります。スコアは、いくつかのシグナルの重み付けされた組み合わせです。

| 信号               | 説明                                                                                                                                 |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------ |
| アカウントの年齢   | 新しいアカウントほどリスク スコアが高くなります。                                                                                    |
| カルマ             | コミュニティのカルマが蓄積されるとリスクが軽減されます。                                                                             |
| 著者の評判         | バックグラウンド ネットワーク インデクサーによって収集された評判データ。                                                             |
| コンテンツ分析     | テキストレベルのヒューリスティック (リンク密度、既知のスパム パターンなど)。                                                         |
| 速度               | 同じ投稿者からの急速な連続投稿はリスクを高めます。                                                                                   |
| IPインテリジェンス | 国レベルの地理位置情報と脅威フィードの検索。国コードのみが保存されます。生の IP アドレスがコミュニティと共有されることはありません。 |

## 階層のしきい値

リスク スコアは、次に何が起こるかを決定する 4 つの構成可能な層のいずれかにマップされます。

1. **Auto-accept** -- スコアが十分に低いため、パブリケーションは問題なく承認されます。
2. **OAuth-sufficient** -- 続行するには、作成者は OAuth 検証を完了する必要があります。
3. **OAuth-plus-more** -- OAuth だけでは十分ではありません。追加の検証 (CAPTCHA など) が必要です。
4. **自動拒否** -- スコアが高すぎます。パブリケーションは完全に拒否されます。

すべてのしきい値はコミュニティごとに構成可能です。

## チャレンジ フロー

パブリケーションが検証が必要な層に分類されると、チャレンジ フローが開始されます。

1. 作成者は、最初に **OAuth** (GitHub、Google、Twitter、およびその他のサポートされているプロバイダ) 経由で認証するよう求められます。
2. OAuth のみの場合不十分な場合（階層 3）、Cloudflare Turnstile を利用した **CAPTCHA フォールバック**が表示されます。
3. OAuth ID は検証のみに使用され、コミュニティや他のユーザーと**共有されることはありません**。

## API エンドポイント

### `POST /evaluate`

リスク評価のためにパブリケーションを送信します。計算されたリスク スコアと必要なチャレンジ層を返します。

### `POST /challenge/verify`

完了したチャレンジの結果 (OAuth トークン、CAPTCHA ソリューション、またはその両方) を検証のために送信します。

### `GET /iframe/:sessionId`

指定されたセッションに適切なチャレンジ UI をレンダリングする埋め込み可能な HTML ページを返します。

## レート制限

レート制限は、著者の年齢と評判に基づいて動的に適用されます。新人または評判の低い著者にはより厳しい制限が課せられますが、確立された著者にはより寛大な制限が適用されます。これにより、信頼できる参加者にペナルティを与えることなく、スパムの洪水を防ぎます。

## バックグラウンド ネットワーク インデクサー

サーバーは、ネットワークを継続的にクロールして作成者の評判データを構築および維持するバックグラウンド インデクサーを実行します。このデータはリスク スコアリング パイプラインに直接入力され、システムはコミュニティ全体で繰り返し善意の参加者を認識できるようになります。

## プライバシー

スパム ブロッカーは、プライバシーを念頭に置いて設計されています。

- OAuth ID はチャレンジの検証のみに使用され、コミュニティに**決して開示されることはありません**。
- IP アドレスは **国コードのみ**に解決されます。生の IP は保存または共有されません。

## データベース

サーバーは、レピュテーション データ、セッション状態、構成のローカル永続化に **SQLite** (`better-sqlite3` 経由) を使用します。

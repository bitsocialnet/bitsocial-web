# อัตลักษณ์และความเป็นเจ้าของชุมชน

โปรไฟล์และชุมชนใน Bitsocial ถูกควบคุมโดยคู่คีย์ ไม่ใช่โดยบัญชีแพลตฟอร์มที่เพิกถอนได้

ที่เปลี่ยนแปลงความสัมพันธ์ระหว่างความเป็นเจ้าของและผู้ให้บริการ คุณสามารถมอบหมายสถานะการออนไลน์ โฮสติ้ง หรือความสะดวกสบายให้กับบริการอื่นได้โดยไม่ต้องให้การดูแลบริการนั้นอย่างถาวรเหนือข้อมูลประจำตัวหรือชุมชนเอง

## เหตุใดกุญแจจึงมีความสำคัญที่นี่

บัญชีแพลตฟอร์มที่โฮสต์จะถูกเช่าในที่สุด มันคงอยู่ตราบใดที่เวทียังคงให้เกียรติมัน

ข้อมูลระบุตัวตนที่ควบคุมด้วยคีย์แตกต่างออกไป:

- คีย์ส่วนตัวเป็นแหล่งที่มาของสิทธิ์
- สามารถเปลี่ยนโฮสติ้งได้
- ตัวเลือกแอปสามารถเปลี่ยนแปลงได้
- ความเป็นเจ้าของไม่จำเป็นต้องย้ายไปพร้อมกับโครงสร้างพื้นฐาน

## การมอบหมายควรหมายถึงอะไร

การมอบหมายควรหมายถึงความช่วยเหลือในเรื่องความพร้อม ไม่ใช่การยอมจำนนต่อการควบคุม

ตัวอย่างเช่น RPC สาธารณะหรือบริการโฮสติ้งสามารถเก็บโหนดออนไลน์ได้ แต่บริการดังกล่าวไม่จำเป็นต้องมีอำนาจในการยึดโปรไฟล์หรือชุมชนอย่างถาวร ความช่วยเหลือนั้นมีประโยชน์ การดูแลควรยังคงเป็นทางเลือก

## จุดที่กว้างขึ้น

สิ่งนี้ทำให้อัตลักษณ์ของ Bitsocial รู้สึกใกล้ชิดกับทรัพย์สินที่ควบคุมด้วยกระเป๋าเงินมากกว่าชื่อผู้ใช้ที่เช่าจากบริษัท ชั้นบริการสามารถสลับออกได้ ความสัมพันธ์แบบเจ้าของควรจะคงอยู่ต่อไปในการแลกเปลี่ยนนั้น

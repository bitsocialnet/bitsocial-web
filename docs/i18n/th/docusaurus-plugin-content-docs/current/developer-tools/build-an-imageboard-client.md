# สร้างไคลเอนต์อิมเมจบอร์ด

ระยะที่ 1 ไม่ใช่แอปอย่างเป็นทางการเพียงแอปเดียวที่ครอบคลุมทั้งหมวดหมู่ 5chan คือข้อพิสูจน์ข้อแรก แต่เป้าหมายที่แท้จริงคือระบบนิเวศของบอร์ดรูปภาพแบบกว้าง: ไคลเอนต์ Bitsocial หลายตัวที่มีภาษาภาพที่แตกต่างกัน ค่าเริ่มต้นของการดูแล โมเดลไดเร็กทอรี และชุมชนเป้าหมาย

## สิ่งที่เฟส 1 ต้องการ

- ไคลเอนต์สไตล์ 4chan ที่คุ้นเคยสำหรับการเริ่มต้นใช้งานกระแสหลัก
- ลูกค้าที่ได้รับแรงบันดาลใจจาก Altchan ด้วยวัฒนธรรมที่แตกต่างและชุดบอร์ด
- ไคลเอนต์ที่เน้นมือถือเป็นหลักหรือแบนด์วิธต่ำ
- ลูกค้าชุมชนเดียวหรือลูกค้าเฉพาะกลุ่มที่มีค่าเริ่มต้นที่ดื้อรั้นสูง
- การกลั่นกรอง สื่อ การเริ่มต้นใช้งาน หรือกระแสการค้นพบที่ดีกว่าแอปแรกที่มาพร้อมกับ

## วิธีที่เร็วที่สุดในการช่วยเหลือ

หากคุณต้องการเส้นทางที่สั้นที่สุดในการจัดส่ง ให้สนับสนุน 5chan โดยตรงก่อน:

- สำรวจแอปถ่ายทอดสดได้ที่ [5chan.app](https://5chan.app)
- ตรวจสอบแหล่งที่มาได้ที่ [github.com/bitsocialnet/5chan](https://github.com/bitsocialnet/5chan)
- เข้าร่วมแชทผู้สร้างที่ [t.me/fivechandev](https://t.me/fivechandev)

## สร้างลูกค้าของคุณเอง

หาก 5chan ไม่ตรงกับชุมชนหรืออินเทอร์เฟซที่คุณต้องการ ให้สร้างไคลเอนต์แยกต่างหากแทน ไคลเอนต์ Bitsocial ที่เข้ากันได้สามารถแชร์เครือข่ายเดียวกันโดยไม่ต้องแชร์การตัดสินใจเกี่ยวกับผลิตภัณฑ์เดียวกัน

1. เรียนรู้เครื่องมือที่ต้องเผชิญโปรโตคอล:
   - [ ดัก Bitsocial React](../react-hooks/)
   - [Bitsocial CLI](../cli/)
2. ตัดสินใจว่าจริงๆ แล้วคุณกำลังสร้างอิมเมจบอร์ดประเภทใด
เลือกโครงสร้างบอร์ด การสันนิษฐานเกี่ยวกับอัตลักษณ์ โมเดลการดูแล โฟลว์การค้นพบ และภาษาภาพก่อน
3. เลือกเส้นทางการนำไปปฏิบัติที่เหมาะกับโครงการ
Fork 5chan หากคุณต้องการเคลื่อนที่อย่างรวดเร็วด้วยฐานอิมเมจบอร์ดที่คุ้นเคย เริ่มต้นใหม่หาก UI หรือโมเดลการโต้ตอบจำเป็นต้องแตกต่างอย่างสิ้นเชิง
4. จัดส่งรุ่นแรกแบบแคบ
ลูกค้ารายหนึ่งที่ให้บริการชุมชนที่แท้จริงแห่งหนึ่งอย่างดีนั้นมีคุณค่ามากกว่าการโคลนนิ่งที่คลุมเครือซึ่งมุ่งสร้างความพึงพอใจให้กับทุกคน
5. เผยแพร่ผลลัพธ์และให้ชุมชนทดสอบ
Bitsocial ปรับปรุงเมื่อผู้สร้างภายนอกส่งลูกค้าที่เอาแต่ใจตัวเองซึ่งแข่งขันกับคุณภาพของผลิตภัณฑ์ แทนที่จะรอให้แอปอย่างเป็นทางการแอปเดียวทำทุกอย่าง

## หลักการออกแบบ

Bitsocial ไม่ชนะการมีลูกค้าที่ได้รับพรเพียงรายเดียว มันจะชนะเมื่อลูกค้าจำนวนมากสามารถอยู่ร่วมกัน แยก เชี่ยวชาญ และตอบสนองความต้องการที่แอปแรกไม่เคยทำได้
